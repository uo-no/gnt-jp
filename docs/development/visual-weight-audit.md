# UX-11-0 — Visual Weight Audit

**実施日**: 2026-07-15  
**種別**: 監査のみ（実装変更なし）  
**対象**: button / chip / badge / sidebar component / Discovery Lobby との整合性

---

## 評価軸の定義

| 指標 | 内容 |
|------|------|
| 視覚重量（現在） | 1〜10。色・塗り・サイズ・繰り返し頻度の合計的な「目の重さ」 |
| 重要度（本来） | 1〜10。この要素がユーザーの判断に寄与すべき度合い |
| ギャップ | 現在 − 本来。プラスは「重すぎ」、マイナスは「軽すぎ」 |

ギャップが ±3 以上を要対処とする。

## 1. ボタン（Button）

### 1-A. `.run-btn` — 「検索する」

| 項目 | 値 |
|------|----|
| 場所 | サイドバー常時表示 |
| スタイル | `background: var(--highlight)`（スチールブルー）塗りつぶし、白文字、`font-weight: 700`、`width: 100%`、`padding: 13px 0`、検索アイコン付き |
| 視覚重量（現在） | **10 / 10** |
| 重要度（本来） | **2 / 10** |
| ギャップ | **+8（重大な過剰）** |
| 判定 | **REDUCE** |

**問題の構造**

Enter キーで検索できる。つまりこのボタンは「Enterを知らないユーザーのフォールバック」にすぎない。  
にもかかわらず、ページ上で最も高い視覚重量を持つ要素になっている。  
ユーザーが最初に見るのは入力欄でも Discovery Lobby でもなく、この青いボタン。  
「このツールはフォームである」という誤ったメッセージを、開いた瞬間に送っている。

**あるべき姿**

- 塗りつぶしを廃止。ghost（枠のみ）またはテキストリンク相当
- 高さを半分以下（`padding: 6px 0`相当）に縮小
- font-weight を 400〜500 に下げる
- ホバー時に初めて色が乗る程度の控えめな存在感

### 1-B. `.hit-open-btn-lg` — 「本文で読む」

| 項目 | 値 |
|------|----|
| 場所 | ヒットカードのフッター（結果件数分、繰り返し出現） |
| スタイル | `background: var(--highlight)`、白文字、`font-weight: 700` — `.run-btn` と**同一色・同一スタイル** |
| 視覚重量（現在） | **8 / 10 × N件** |
| 重要度（本来） | **5 / 10**（ユーザーの最終ゴールに近いが、結果を読んでから判断するアクション） |
| ギャップ | **+3（過剰、かつ繰り返しによる総量が深刻）** |
| 判定 | **REDUCE** |

**問題の構造**

631件の結果に対して、631個の「本文で読む」ボタンが並ぶ。  
さらに `.run-btn`（検索する）と同一の `var(--highlight)` 塗りを使っているため、  
「検索を実行するアクション」と「結果を開くアクション」が視覚的に等価に見える。  
Apple Books のリストでは、行タップ全体がアクション。個別の filled CTA はない。

**あるべき姿**

- ghost（枠のみ）または非常に薄い tinted ボタン（`background: var(--highlight-light)` 相当）
- font-weight: 500 以下
- カード全体をクリッカブルにすることで、ボタン自体の主張を下げる方向も検討対象

### 3-A. `.ht-badge` — ヒットタイプバッジ（「語根」「フレーズ」等）

バッジが意味を持つのは、複数の検索モードが同時に発火した時（lemma + phrase が混在する結果等）のみ。

## 5. Discovery Lobby との視覚的一貫性

### 5-A. Discovery Lobby の世界観

Discovery Lobby は現在このプロダクトの中で最も完成度が高い部分。

| 要素 | 視覚重量 | 手段 |
|------|---------|------|
| タイトル「今日は、どんなことばを読みますか？」 | 2 | `color: var(--text-sub)`、`font-weight: 400` |
| テーマ語4件 | 3 | `font-family: 'Gentium Plus', serif`、余白、塗りなし |
| 背景 | 0 | 純白 |

**この世界観の定義:**  
- 主張するのはコンテンツ（テキスト）のみ
- 色・塗り・枠は使わない
- 重量の差はタイポグラフィと余白で表現する

### 5-B. サイドバーとの断絶

サイドバーの視覚言語は Discovery Lobby と真逆。

| 要素 | 視覚重量 | 乖離 |
|------|---------|------|
| `.run-btn`（「検索する」） | 10 | Discovery Lobby のどの要素より重い |
| `.layer-row.active-lemma` | 7 | 青ティント・青枠・青テキストで存在を主張 |
| タイトル「聖書原文検索」 | 3 | テーマ語より軽い |

**ユーザーの体験:**  
Discovery Lobby の静寂（右パネル）→ サイドバーを見る → 突然の視覚的ノイズ  
この断絶が「何か業務アプリみたい」という印象を作っている。

### 5-C. 検索結果と Discovery Lobby の一貫性

| 要素 | 状態 | 整合 |
|------|------|------|
| ヒットカードのテキスト（聖書本文） | セリフ体 `Gentium Plus` | ✅ Discovery Lobby と同一フォント |
| 節番号（`.hit-ref`） | `color: var(--highlight)` 青リンク | △ 機能的には正しいが、色使いが多くなる |
| `hit-open-btn-lg`（本文で読む） | 塗りつぶし青 | ❌ Discovery Lobby に存在しない「重さ」が突然出現 |
| Concept Insight カード | グレー背景 `var(--bg-panel)` | △ Discovery Lobby の純白と対比が強い |
