# bible_data.japanese ポリシー(Stage H-3b)

策定日: 2026-07-18
位置づけ: bible_data.japanese が担うべき責務を正式に定義する設計文書。
Stage H-2(μόδιος の単純誤訳修正)と Stage H-3a(疑問詞・関係詞 11 lemma の実測監査)を
根拠とする。**本文書はポリシー策定のみ。データ・コード・生成物は一切変更しない。**

---

## 1. 目的

**bible_data.japanese は翻訳文ではない。**

役割は次の一点に限定される:

> **Reading Engine が形態・構文・意味情報と組み合わせて読解表示を生成するための、
> 安定した「代表語」を提供すること。**

bible_data.japanese は最終的な訳文でも、文脈に最適化された訳でもなく、lemma レベルの
基本的な意味を表す**安定した代表語**である。文脈・形態・構文による変化は Engine 以降の
層が担う。

---

## 2. 責務境界

| bible_data.japanese が担当する | bible_data.japanese が担当しない |
|---|---|
| 安定した代表語 | 文脈解釈 |
| lemma レベルの基本意味 | gender による変化 |
| — | syntax による意味決定 |
| — | 語用論的ニュアンス |
| — | 節個別の訳し分け |

原則: **「一つの lemma に一つの安定した代表語」**。その代表語が形態や文脈で変化する部分は
bible_data ではなく Engine / Semantic / Presentation が担う。

---

## 3. Type 分類

Stage H-3a の実測(NT 全巻)に基づき、lemma を 3 型に分類する。

### Type A — 固定日本語を持ってよい語

条件(すべて満たす):
- 意味が安定している
- 形態変化で意味が変わらない
- 文脈依存が低い(実測 unsafe 率がごく低い)

例(Stage H-3a 実測):

| lemma | Strong | 固定訳 | unsafe 率 |
|---|---|---|---|
| πῶς | G4459 | どのように | 0.0% |
| πόσος | G4214 | どれほど | 0.0% |
| ποῦ | G4226 | どこに | 3.9% |

### Type B — 代表語として保持する語

条件:
- lemma 見出しとして有用
- ただし本文表示では Engine 補正がありうる

例:

| lemma | Strong | 固定訳 | unsafe 率 |
|---|---|---|---|
| ποῖος | G4169 | どのような | 3.0% |
| ὁποῖος | G3697 | どのような | 0.0% |
| πόθεν | G4159 | どこから | 13.8% |
| πότε | G4219 | いつ | 21.1% |

**注意: Display Label(見出し)と Reading Output(本文表示)は別責務。**
Type B は Display Label としては固定訳で十分だが、Reading Output では形態・文脈による
補正の余地がある。

### Type C — 固定日本語を禁止する語

条件(いずれか):
- gender 依存
- case 依存
- syntax 依存
- discourse 依存

例(Stage H-3a 実測):

| lemma | Strong | 現在の固定訳 | 禁止理由 | unsafe 率 |
|---|---|---|---|---|
| τίς | G5101 | 誰 | gender 依存(男性=誰 / 中性=何・なぜ / 女性=何・どれ) | 74.4% |
| ὅς | G3739 | 〜する者 | 人だけでなく物・事も対象(中性 482 件)。関係節構文 | 23.8% |
| ὅστις | G3748 | 〜する者 | 人限定訳では不十分。関係詞 | 4.3% |
| ὅσος | G3745 | 〜するだけ | 数量・程度・関係表現で変化(中性 72 件) | 15.3% |

**Type C は「単一の固定訳では一定割合で誤る」語であり、bible_data は安定代表語を保持しつつ、
実際の変化は Engine(形態)・Semantic(文脈)が担う。**

---

## 4. 層別責務

| 層 | 責務 |
|---|---|
| bible_data.japanese | 安定代表語 |
| Morph Engine | gender・number・case・person による形態変化 |
| Syntax Engine | 関係詞・疑問詞・修飾関係の構造反映 |
| Semantic Layer | 文脈による語義選択 |
| Presentation | 表示形式のみ |

この分離により、Type C の lemma でも「bible_data は一つの代表語を持ち、Engine が形態で
変化させ、Semantic が文脈で選択し、Presentation が表示を整える」形で責務が明確になる。

---

## 5. 禁止事項

bible_data.japanese に入れてはいけないもの:

- 辞書説明文
- 長い語義説明
- 文脈限定訳
- 神学的解釈
- 特定節だけに適用する訳

### 例

| | 例 |
|---|---|
| **禁止** | 「内なる思考を表すもの」(λόγος の辞書説明文) |
| **許可** | 「ことば」(λόγος の安定代表語) |

| | 例 |
|---|---|
| **禁止** | 「姦淫の女」(μόδιος の誤訳・全く別語) |
| **許可** | 「枡」(μόδιος の代表語) |

説明文・文脈限定訳は Display Label(見出し)でも Reading Output でもなく、③語義説明
(gloss_ja / abbottSmith)の領域である(docs/display-label.md の 3 層分離)。

---

## 6. 今後の監査ルール

新しい lemma を追加・修正する場合、以下を確認する:

1. **固定訳で全形態をカバーできるか**
   — できないなら Type C(Engine 補正が必要)
2. **gender / case で意味変化しないか**
   — 変化するなら Type C
3. **Engine で補正すべき領域ではないか**
   — 形態変化・文脈選択は bible_data の責務ではない

判定フロー:

```
新 lemma の japanese 候補
   ↓
全形態で意味が一定か? ── No ──▶ Type C（安定代表語のみ・変化は Engine/Semantic）
   ↓ Yes
文脈依存が低いか? ──────── No ──▶ Type B（見出し可・本文は Engine 補正）
   ↓ Yes
Type A（固定可）
```
