# 関係詞 Syntax Phase 3 責務設計(Stage K-5)

策定日: 2026-07-20
位置づけ: 関係詞の**語順再構成**について、**Syntax と PhraseRenderer の責務境界を定義**する設計文書。
K-4 Phase 2(relativeSyntax の構造情報保持・整合確認)で確定した「role→役割は語順で表現=Phase 3」を

## 0. 実測(設計の前提)

Phase 3 は先行詞リンクと語順を扱うため、束縛関係 1,071 の先行詞解決・位置関係を実測した。

- **先行詞の解決**: **referent は token の `verseId`(Macula word-ID: n+book+chap+verse+word)を指す**。
  verseId 索引で **1,006 / 1,071(93.9%)が先行詞トークンに解決可能**(例: ὃ referent=n45001001009 →
  先行詞 εὐαγγέλιον「福音」)。解決不可 65 件。
- **語順**: **先行詞が関係詞より前 975 件(91%)**(Greek 標準: 先行詞 + 関係節)。日本語では節を
  **先行詞の前へ**置く必要がある。先行詞後置 31 件(稀)。
- **先行詞–関係詞の距離**: 1–2(隣接/単純)**535 件(53%)** / 3–5 221 / 6–10 107 / 11+(遠隔/複雑)143。

---

## 1. 設計範囲

- referent に基づく **先行詞リンクの利用**(verseId 解決)。
- 関係詞と先行詞の **構造関係**。
- 関係節の **位置・節境界情報**。
- 日本語語順再構成に必要な **構造情報**。
- **関係節を先行詞の前に置く**ための責務境界。

---

## 2. Syntax 担当(構造情報の保持・提供)

Syntax は **注釈由来の構造的事実を保持・提供**する。語順は変えず、日本語文字列も生成しない。

| Syntax が担当 | 内容 |
|---|---|
| **関係詞と先行詞の構造関係** | どの関係節がどの先行詞を修飾するか(referent リンク) |
| **referent に基づく先行詞リンク** | referent(verseId)を先行詞トークンへ解決し、リンクを提供(索引参照・推論なし) |
| **関係節の節境界・位置情報** | 関係詞トークンの位置・先行詞トークンの位置・(単純節の)節範囲 |
| **role 情報** | K-4 の relativeSyntax(role/referent)を提供 |
| **語順変更をしない** | Syntax は順序を変えない(構造情報を渡すのみ) |
| **日本語表層文字列を生成しない** | 文字列生成は PhraseRenderer |

- referent の解決は **verseId 索引参照**(注釈読み取り)であり、意味推論ではない。
- 節境界の確定は **単純節では注釈・位置から導ける**(§5)。複雑節は範囲外。

---

## 3. PhraseRenderer 担当(語順再構成・表層生成)

| PhraseRenderer が担当 | 内容 |
|---|---|
| **語順再構成** | Syntax 提供の構造情報に基づき日本語語順へ再構成 |
| **修飾節の前置** | 関係節を先行詞の前へ配置(連体修飾化) |
| **関係節と先行詞の表層結合** | 節と先行詞(頭語 者/もの ではなく実先行詞)の表層的結合 |
| **日本語表層文字列の生成** | 再構成後の表層(チップ列)の生成 |

- PhraseRenderer は **Syntax が解決済みの先行詞リンクを受け取る**。**先行詞を推論・生成しない**。
- 既存 core/phrase-renderer.js(Stage A)の枠組みを用いる(本設計は責務のみ・実装しない)。

---

## 4. Syntax → PhraseRenderer 間で渡す構造情報の範囲

Syntax が PhraseRenderer へ渡す情報(構造的事実のみ・文字列なし):

| 情報 | 内容 |
|---|---|
| 先行詞トークン参照 | referent を解決した先行詞トークン(verseId 索引) |
| 先行詞位置 | 先行詞の globalIdx(語順再構成の基準点) |
| 関係詞トークン参照・role・case | 関係詞の位置・節内役割 |
| 節境界(単純節) | 関係節に属するトークン範囲(単純節で確定できる範囲) |
| 頭語 | Morph 出力(者/もの・置換可否の情報) |

- **渡さない**: 助詞文字列・語順の結果・意味解釈・自由関係の判断。
- **referent 利用の責務**: Syntax=解決(verseId→先行詞トークン)・リンク提供。PhraseRenderer=解決済み
  リンクに基づく前置・表層結合。**PhraseRenderer は先行詞の推論/生成をしない**。

---

## 5. 単純な関係節と複雑な関係節の適用範囲

| 区分 | 条件 | 件数(目安) | Phase 3 |
|---|---|---|---|
| **単純関係節** | 先行詞が解決可能・**先行詞前置**・**近接(距離小)**・入れ子なし・連続節 | 距離1–2 で 535(53%)・先行詞前置 975 | **第一次適用範囲** |
| **複雑関係節** | 遠隔先行詞(距離 11+ 143)・**先行詞後置**(31)・入れ子・非連続節・**解決不可**(65) | 約 240+ | **第一次スコープ外**(現状維持) |

- **Phase 3 第一次は単純関係節に限定**(先行詞解決可・前置・近接・非入れ子)。語順再構成の
  正当性・自然性を担保しやすい範囲から着手する。
- **複雑関係節は Phase 3 第一次で扱わない**(現状の逐次表示を維持)。遠隔・後置・入れ子・解決不可は
  再構成の誤り・自然性劣化のリスクが高く、将来段階で慎重に扱う。

---

## 6. Gate(Phase 3 実装開始条件)

| Gate | 判定条件 |
|---|---|
| **G1** | K-4 Phase 2(relativeSyntax 保持)が実装済・凍結されている(充足) |
| **G2** | referent→先行詞の **verseId 索引解決**が用意でき、単純節の対象を切り分けられる(実測 93.9% 解決) |
| **G3** | 単純関係節の **節境界確定**手段が定義されている(先行詞前置・近接・非入れ子) |
| **G4** | PhraseRenderer 協働の枠組み(既存 phrase-renderer.js)で語順再構成が安全に行える見通し |
| **G5** | **出力変化(語順)に対する FROZEN プロトコル**(回帰追加→悪化 0→基準更新)と QA が用意 |
| **G6** | **chip⇔panel 一致 100%・破損形 0** を再構成後も維持できる(flow と panel の整合) |

- G1〜G6 すべて充足で Phase 3 実装(単純関係節)へ進む。複雑関係節は別 Gate。

---

## 7. QA 方針

| 項目 | 方針 |
|---|---|
| 先行詞解決率 | verseId 索引で単純節対象が解決できること(実測ベース) |
| 語順再構成の正当性 | 単純節で節が先行詞の前へ正しく配置されること(構造監査) |
| 自然性(読者評価) | 単純節の再構成後の日本語を Stage F/G の読者評価基準で確認(悪化 0) |
| 悪化 0 | 再構成対象外(複雑節・自由関係・非関係詞)の出力が不変。対象節も自然性が劣化しない |
| chip⇔panel 一致 | 再構成後も 100%・破損形 0 |
| Morph v1・既存回帰 | Morph 出力(者/もの)不変・既存 8 スイート ALL PASS(FROZEN プロトコルで基準更新) |
| 複雑節の非適用 | 複雑節・解決不可 65 が現状維持であることを確認 |

- 測定は NT 全巻・実 FS(サンドボックス不可・重複「 2.json」不在を確認)。

---

## 8. FROZEN 条件

Phase 3 責務設計を FROZEN とする条件:

- Syntax ↔ PhraseRenderer の責務境界が確定している(§2/§3/§4)。
- referent 利用の責務(Syntax=解決・提供 / PhraseRenderer=前置・表層)が確定している。
- 単純関係節と複雑関係節の適用範囲が確定している(§5)。
- Gate(§6)・QA 方針(§7)・扱わない範囲(§9)が定義されている。
- 本設計は責務のみで、実装・語順変更・文字列生成を含まない。

---

## 9. Phase 3 で扱わない範囲(明示)

| 項目 | 理由 |
|---|---|
| **複雑関係節**(遠隔・後置・入れ子・非連続・解決不可 65) | 再構成の誤り・自然性劣化リスク。第一次スコープ外 |
| whoever / whatever / 数量(ὅσος)/ 不定(ὅστις)/ 自由関係(referent 無 587) | **Semantic 領域** |
| adv の意味・助詞判断 | prep 依存・保留 |
| role/referent に基づく**意味の推論** | 禁止(構造の読み取りのみ) |
| **Syntax による語順変更・文字列生成** | 禁止(語順=PhraseRenderer・文字列=PhraseRenderer) |
| **PhraseRenderer による先行詞の推論・生成** | 禁止(Syntax が解決済みリンクを渡す) |
| **Morph Rule Engine v1 の変更** | 禁止(頭語 者/もの・7 lemma は不変) |
