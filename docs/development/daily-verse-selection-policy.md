# 日替わり聖句 選定方針

**フェーズ**: D-1D  
**作成日**: 2026-07-15  
**種別**: 選定ポリシー（実装なし・聖句未選定）  
**参照**: `docs/daily-verse-year-plan.md`, `docs/daily-verse-slots.md`, `docs/daily-verse-schema.md`

---

## 1. 基本方針

### 1-1. 現フェーズ（NT のみ）

- 対象コーパスは **新約聖書（NT）のみ**
- 選定する節はすべて `corpus: 'NT'` とする
- 旧約引用を含む NT 節も採用可（例: マタイ 22:37 は申命記からの引用だが NT テキストとして収録）
- OT 専用箇所（詩篇・イザヤ等）は現フェーズでは選定しない

### 1-2. 将来フェーズ（OT・LXX 拡張）

- `corpus: 'LXX'` エントリを追加するだけで拡張できる設計を維持する
- OT 追加時も `getDailyVerse()` の関数シグネチャは変更しない
- `storyStage` の enum は OT 節が加わる想定で定義済み（`docs/daily-verse-schema.md` 参照）
- 索引の準備（`IDX.l3Ready`）が整ったタイミングで Phase D-5 として着手

### 1-3. 年間構成の原則

- 救済史の流れを一年でたどる（創造→堕落→契約→受肉→受難→復活→聖霊→宣教→終末）
- 人気聖句集・「よく知られた節のまとめ」ではなく、年間を通じたバランスある構成
- 特定の教派・神学傾向に偏らない普遍性を重視する
- 日常の信仰生活に接地した節を優先し、神学的専門語を多用する節は補助的位置づけとする

## 2. 選定原則

### P-1: キリスト中心

- イエス・キリストの人格・働き・教えに関わる節を核とする
- 父なる神、御霊の働きもキリストとの関係において読める節を選ぶ
- 新約の大使命・福音宣言（1コリ 15:3-4 等）を年間に必ず含める

### P-2: 救済史との整合

- 各節が `docs/daily-verse-slots.md` のスロットの `storyStage` と自然につながること
- 歴史的背景（誰に・いつ・なぜ語られたか）を無視した脱文脈引用を避ける
- 文脈から切り離すと意味が逆転する節は除外する（例: 「すべてのことが益となります」を苦難の保証として孤立使用しない）

### P-3: 信仰生活への適用

- 個人の祈り・黙想・日常生活に直接響く節を優先する
- 礼拝・感謝・悔い改め・奉仕・隣人愛など実践的テーマを年間に散りばめる
- 抽象的な神学命題のみを述べる節より、約束・招き・命令の形式の節を重視する

### P-4: 教派的中立性

- 特定教派が強調する神学箇所への過度な集中を避ける
- 洗礼・聖餐・終末論・霊的賜物など教派差異が大きいテーマは複数視点から節を選ぶ
- 信条（ニカイア・使徒信条）の骨格に沿って年間を構成し、偏りの検証基準とする

### P-5: 言語的アクセシビリティ

- 日本語訳として日常語に近い表現の節を優先する
- 意訳と直訳のバランスが取れた翻訳（新改訳 2017・聖書協会共同訳を参照基準とする）
- 表記・読みが平易であること。難読漢字や文語体が多い節はルビ対応するか避ける

### P-6: 検索可能性

- `keywords` に挙げる語がすべて NT 索引で 1件以上ヒットすること（`_exampleSearch()` 検証必須）
- 節に検索できる語が1語も含まれない場合は採用しない
- 抽象概念のみの節（検索キーワードを持ちにくい）は別の節で代替する

## 4. 採用基準

| 基準 | 説明 |
|-----|------|
| A-1 約束 | 神からの約束・保証を含む節（「〜してくださる」「〜があります」等） |
| A-2 招き | 行動への招き・命令を含む節（「来なさい」「愛しなさい」等） |
| A-3 福音宣言 | 救いの核心を述べる節（十字架・復活・恵み・信仰等） |
| A-4 神の性質 | 神・キリスト・御霊の性質や働きを簡潔に述べる節 |
| A-5 弟子道 | 弟子の生き方・使命・倫理を具体的に示す節 |
| A-6 共同体 | 教会・互いへの奉仕・赦し・愛を示す節 |
| A-7 希望 | 終末的希望・永遠のいのち・復活を指し示す節 |
| A-8 祈り | 主の祈り・執り成し・感謝の祈りの模範を示す節 |

## 5. 除外基準

| 基準 | 説明 | 例 |
|-----|------|-----|
| E-1 文脈依存 | 前後の文脈がないと意味が逆転・不明になる節 | 系図・名簿・旅程記録 |
| E-2 論争的 | 特定の教派論争に使われやすく中立性を損なう節 | 霊的賜物の一部・終末の時期計算 |
| E-3 倫理的難節 | 現代の倫理観と強い摩擦を生む節（説明なしに単独提示が困難） | — |
| E-4 主語不明 | 主語・対象が不明で孤立使用が難しい節 | 議論の中間節・間接話法の断片 |
| E-5 検索不能 | keywords に使える語が NT 索引に1語も存在しない節 | （検証後に判定） |
| E-6 長すぎる | 節単体が 5行を超え、画面上の表示として不適切なもの | 長い比喩・長い系図節 |
| E-7 重複過多 | 共観福音書でほぼ同一内容の節を複数採用する場合（片方に絞る） | マタイ5:3 ≒ ルカ6:20 等 |

## 6. バランス基準

### 6-1. 同一書巻

| 規模 | 同一書巻上限 |
|-----|------------|
| 100節 | 1書巻 ≤ 20節（20%） |
| 365節 | 1書巻 ≤ 55節（15%） |

例外: ヨハネ福音書は「わたしは…」宣言の豊富さから 365節で最大 55節まで許容。

### 6-2. 同一章

| 規模 | 同一章上限 |
|-----|----------|
| 100節 | 1章 ≤ 4節 |
| 365節 | 1章 ≤ 6節 |

山上の垂訓（マタイ 5〜7章）・ヨハネ 14〜17章など密度の高い章は例外的に最大 8節まで許容するが、連続配置しない。

### 6-3. 同一テーマ

| 指標 | 上限 |
|-----|------|
| 同一 `storyStage` の連続日数 | ≤ 10日 |
| 年間で同一 `storyStage` の総割合 | ≤ 25%（≈ 91日） |
| 「愛」を主テーマとする節の連続日数 | ≤ 7日 |

### 6-4. 同一著者

| 著者 | 365節上限 | 備考 |
|-----|----------|------|
| ヨハネ（福音書＋書簡＋黙示録） | ≤ 80節（22%） | |
| パウロ書簡（全13書） | ≤ 130節（36%） | 書巻別上限は 6-1 参照 |
| ルカ（福音書＋使徒） | ≤ 65節（18%） | |

## 7. データ設計方針

### 7-1. corpus フィールドの扱い

- フィールドは **すべてのエントリに必須** とする（省略不可）
- 現フェーズでは全エントリに `corpus: 'NT'` を付与する
- 将来 OT を追加する場合は `corpus: 'LXX'` エントリを追加するのみでよい
- UI 側（`renderDiscoveryLobby`）は corpus 値を見て検索対象インデックスを切り替える設計を予定（Phase D-4 実装時）

### 7-2. NT 索引の制約

- `IDX.ready === true` のとき NT が利用可能
- `IDX.l3Ready === true` のとき NT + LXX + 日本語が利用可能
- `corpus: 'LXX'` のエントリは `IDX.l3Ready` が true のときのみ keywords 検索が機能する
- 現フェーズでは `corpus: 'NT'` のみのため `IDX.ready` チェックで十分

### 7-3. storyStage の将来拡張

`docs/daily-verse-schema.md` に定義済みの enum に OT 用を追加できる設計を維持する。

**現行 enum（NT 対応済み）**:

```
創造 / 堕落 / 契約 / 出エジプト / 預言 / 受肉 / 律法の成就 /
律法と知恵 / 受難と贖い / 復活 / 新創造 / 聖霊と教会 /
宣教 / 信仰の旅路 / 終末的希望
```

**OT 追加時に拡張する候補（参考）**:

```
族長時代 / 出エジプト / 荒野 / 征服と定住 / 王国時代 /
分裂王国 / 捕囚 / 帰還と再建 / 詩篇の礼拝 / 知恵と創造神学
```

NT 節のスロットで使用する `出エジプト`（型）の記法はOT追加時に整理する。

### 7-4. OT 追加時の移行手順（参考・将来用）

1. `IDX.l3Ready` が真になる前提を確認
2. `storyStage` enum に OT 用値を追加（`docs/daily-verse-schema.md` 改訂）
3. OT 節エントリを `DAILY_VERSES_365` に追加または別配列で管理
4. `getDailyVerse()` の corpus チェックロジックを追加
5. Phase D-3 相当の keywords 検証（LXX 索引）を実施

### 8-3. 自動検証（Phase D-3 実施時）

検証スクリプトが確認する項目:

```
1. 配列長 === 365
2. 各エントリの必須フィールド完全性
3. corpus が 'NT' | 'LXX' のいずれか
4. _exampleSearch(keyword) で全 keyword が 1件以上ヒット
5. storyStage が定義済み enum 内
6. liturgicalTag が定義済み enum 内
7. season が定義済み enum 内
8. 同一 ref の重複 ≤ 3件
9. storyStage 連続 ≤ 10日
```
