# UX-11-2 — Visual Rhythm Audit

**実施日**: 2026-07-15  
**種別**: 監査のみ（実装変更なし）  
**対象**: Discovery Lobby 状態 / 検索結果状態のリズムパターン

---

## 評価軸の定義

### Weight（視覚重量）

`0〜10` で数値化。visual-weight-audit.md と同一スケール。

### Tempo（視覚テンポ）

| Tempo | 意味 | 特徴 |
|-------|------|------|
| **Fast** | 目が素通りできない | 高コントラスト・塗りつぶし・動き・アイコン |
| **Medium** | 一拍止まる | border あり・グレー背景・テキスト+機能 |
| **Slow** | ゆっくり読む | 低コントラスト・余白・セリフ体・小さい |
| **Silent** | 目が休む | 空白・意図的な無 |

### 理想リズムの原則（Apple 準拠）

```
Slow → Slow → Medium → Slow → Medium → Slow
```

速いテンポの要素は1画面に**1つまで**。  
連続する Fast は「叫ぶ文章」と同じ——読む気を失わせる。

**問題のパターンが2種類:**

1. **水平 Fast の連打**（行4-5）: 概念チップ 4個が横に並ぶ = 横方向の Fast ×4

## 4. Apple が重視するリズムの原則

> **「強調は比率によって生まれる。すべてが強調されると、何も強調されない。」**

現状のこのプロダクトは、「検索する」「本文で読む」の二つを強調している。  
しかしその二つが**常に存在する**ため、強調が無意味化している。

Apple の実装で参考になる事例:
- **Apple Music のアルバム画面**: 再生ボタンは塗りつぶし。他のすべてのアクション（シャッフル、追加、...）は ghost またはアイコンのみ。Primary が1つだから際立つ。
- **Safari のアドレスバー**: 入力前は薄いグレー。タップしてフォーカスされた時だけ存在感を持つ。
- **Spotlight**: 検索ボタンがない。入力が行動を起こす唯一のトリガー。
