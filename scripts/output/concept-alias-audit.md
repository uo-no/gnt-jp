## 6. Phase 8-2 Recommendation

### 追加推奨リスト（Priority 1）

```json
faith:         ["忠実な", "忠実"]
grace:         ["賜物"]
hope:          ["希望する"]
gospel:        ["福音を伝える"]
perseverance:  ["忍耐する"]
eternal-life:  ["永遠のいのち"]
```

**追加後:** 50 → 57 aliases（7 追加）

### 優先度の判断根拠

1. **faith + 忠実な/忠実**: πιστόs は NT で「忠実な」意味が 67 例と頻出。信仰の概念が「神への忠実さ」を内包する以上、これは概念の中心入口として機能する。
2. **grace + 賜物**: χάρισμa を「賜物」と呼ぶ文化的文脈（カリスマ運動系の読者）を考えると、恵みへの入口として実用性が高い。
3. **hope + 希望する**: 動詞形の希望を持つことを探したい読者に向けて。31 例で実用的。
4. **gospel + 福音を伝える**: εὐαγγελίζω は 54 例と高頻出。「福音を伝える使命」を学ぶ読者が自然に入力する。
5. **perseverance + 忍耐する**: 動詞形補完。「忍耐する者は救われる」等の文脈での検索を想定。
6. **eternal-life + 永遠のいのち**: 日本語訳聖書の標準表記揺れ。現行 label「永遠の命」と区別する意味での補完。

### Priority 2（Phase 8-3 以降で検討）

```json
holy:        ["聖別する"]   // 28例・高信頼だが「聖化」alias との差分を確認してから
holy-spirit: ["霊的な"]     // 26例・意味範囲を神学的に評価してから
```

### 変更しないもの

- `霊` (holy-spirit): 高信頼基準を満たさない（Phase 6-1 と同方針を維持）
- `後悔する` (repentance): 神学的に μετανοέω と μεταμέλομαι を混同させる
- `赦す` (grace): 救い・罪の領域と重複する
- `天国` (kingdom): Phase 6-1 の判断を維持
- `義と認める` (righteousness): 既存「義認」alias で代替

## Notes

- `lemma_dict.json` の japanese フィールドには「読み込んでいます...」(placeholder)が多く残っている。実際の gloss 集計は `bible_data/nt/*/` トークンの `japanese` フィールドを使用した（137,741 トークン走査）。
- alias は検索テキストのマッチング入口であり、alias の語そのものをテキスト検索するわけではない（→ concept の terms/lemma を展開する）。したがって alias の採用基準は「その語が当該概念に読者を正しく導くか」であり、NT テキストでの出現頻度とは必ずしも一致しない。
- `愛情`・`良い知らせ` のように NT テキストに直接出現しない alias も、概念への自然な日本語入口として有効（教育的入口としての役割）。
