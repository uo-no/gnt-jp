# StudyPanel Reading Philosophy Audit（Phase 23.6）

- 監査日: 2026-07-07
- 目的: Phase 23.5 の設計理念を精密化し、**「StudyPanel は理解を提供する装置ではなく、本文を読むための補助装置である」**ことを正式に定義する
- 対象: 設計文書のみ（docs/README.md・studypanel-design-principles.md・Phase 21–23/22.5 の各監査文書）。コード・Registry・テスト・UI 変更ゼロ
- 本フェーズの処置: 正典（studypanel-design-principles.md）への追補 2 箇所 + 本監査書の新設

## 1. 現行思想の評価

Phase 23.5 で確定した理念（読書支援・Wallace = 内部知識ベース・本文→意味→まとまり→流れ→理解）は本フェーズの焦点と矛盾しない。ただし横断確認の結果、**「読者を本文へ戻す」という帰還の原則はどの文書にも明文化されていなかった**（「本文へ戻す/代行しない/commentary ではない」に相当する記述は検索で 0 件）。つまり現行思想は「本文から出発する」ことは定めていたが、「本文へ帰着する」ことを定めていない — 説明が良質になるほど**パネル内で完結し読書を代替する**危険（注解書化）が未然に塞がれていなかった。本フェーズでこの欠落を原則 1 として正典に追補した。

## 2. 本文中心性 Audit（Audit 1）

**判定: ✓ 合格。** 正典 §1 は「本文 → 意味 → まとまり → 流れ → 理解」を明記し、Phase 22 の推奨アーキテクチャ図も Text を頂点に置く。「StudyPanel → 解説 → 本文理解」という逆向きの思想（解説が入口になる構図）は全設計文書に存在しない。README も「言語の壁に気づかせない。読書の流れが途切れない」を中核に据えており一致。

## 3. 説明代替化 Audit（Audit 2）

**判定: 現行文書に違反なし。ただし将来リスク 2 点を特定し、正典追補で塞いだ。**

- 現行の Reading Note テンプレ（47 文）は 1 文・要約的で、注解書・翻訳解説・文法講義のいずれにもなっていない（むしろ内容が薄い側 — Phase 22.5 Audit 4）
- **リスク 1**: 正典 §5 の旧文言「結論のみを言う」は、改善が進んだ場合に「結論を渡して終わる」= 読まなくても済む説明へ漂流し得る → **追補**: 「結論は本文へ戻す形で言う。説明で完結させない」
- **リスク 2**: Phase 22.5 の「意味の即答」（Step 3）は、即答が十分に良質になった場合に本文への帰還動機を失わせ得る → 原則 1 の判定基準（「読み終えたら本文のどこを見ればよいか」が判る形で終わる）を全説明の合格条件として明文化
- StudyPanel が本文を読まなくても済む設計になっている箇所: 現状は説明が乏しいため事実上存在しない。**代替化は現在の欠陥ではなく、改善フェーズで生まれる欠陥**である — だからこそ実装前の今、原則として固定した

## 4. Wallace 位置づけ Audit（Audit 3）

**判定: ✓ 合格。** 「Wallace を学ぶ」に類する表現は全文書で 0 件（Phase 23.5 の横断検索を本フェーズで再確認）。正典 §3 は「Wallace = 教師ではなく、読書メモを生成するための内部知識ベース」「Wallace-based の意味は『Wallace の統語論で読みの根拠を保証する』」と定義済みで、本フェーズの基準（OK: 「Wallace を根拠として本文理解を補助する」）と完全に一致する。

## 5. 文法提示順序 Audit（Audit 4）

**判定: 方向は確定済み・具体基準が未成文だった → 追補で成文化。**

- 正典 §4 は「型名（label_ja）は回答の冒頭に置かない。名前は再会と気づきの後」と定めており、順序思想（本文の観察 → 意味 → 必要なら文法名称）は既に存在した
- ただし文レベルの判定基準（NG/OK の形）は未成文だった → **追補**（正典 §5）:
  - NG: 「これは主語的属格です。」
  - OK: 「『神の愛』は『神が愛する愛』として読むことができます。文法では主語的属格と呼ばれます。」
- この基準は将来のテンプレ執筆（Phase 21 Top20 の個別型など）の**検収条件**として機能する

## 6. Reading Note 品質基準（Audit 5）

Reading Note が満たすべき基準を確定する:

| 基準 | 現状 | 判定 |
|---|---|---|
| 本文の言葉から説明している | テンプレはカテゴリ記述（「目的が示されています」）で本文の言葉を含まない | **✗ 未達**（Phase 22.5 Audit 4 で記録済み・設計改善点） |
| Q1: 何と言っているのか | グロス（見出し語+訳）のみが回答。Reading Note 自体は答えていない | △ 部分 |
| Q2: なぜこの訳になるのか | 未回答（242 型が素材のまま未接続） | ✗ 設計改善点 |
| Q3: どこまでが一つのまとまりか | 未回答（Phrase 空洞） | ✗ 設計改善点 |
| Q4: 前にも見た構造か | 未回答（再会の概念なし） | ✗ 設計改善点 |

**4 問への回答責任を Reading Note（と周辺層）に正式に割り当てる**: Q1 = Reading Note（本文の言葉で）/ Q2 = 求められたときの理由層 / Q3 = かたまり提示 / Q4 = 再会設計。いずれも Phase 22.5 Audit 10 の Critical/High と同一の台帳項目であり、新規課題は生じない — 本フェーズは責任の割当を成文化した。

## 7. Observation 基準（Audit 6）

**判定: ✓ 合格 — さらに、Observation は原則 1 の最も直接的な実装であると位置づけを強化。**

正典 §6 は「文法知識の提示ではなく、理解を助けるための『見る場所の指さし』」と定義済み。本フェーズの観点を加えると、Observation は本質的に**読者の目を本文へ戻す装置そのもの**である（「ἵνα の後の動詞の形を見て」は、パネルから本文への帰還命令）。したがって Observation は原則 1 の模範であり、逆に「見なくても分かる要約」を Observation と呼ぶことは定義違反となる。

## 8. UI 思想との整合（Audit 7・実装は対象外）

設計上の段階配置を確認: 初期表示 = 本文理解（Reading Note 1 文・用語ゼロ）/ 追加表示 = 観察（2 回目以降・求めたとき）/ 詳細表示 = 文法・Wallace（名前 → 説明 → 別解 → 原典）。これは Phase 22 の Layer 1–4・Phase 22.5 の 10 段プロセスと同一構造であり、**設計文書間の矛盾なし**。現行実装が この配置に達していないこと（文法表・英語辞書が前面）は既知の接続課題（Phase 22.5 Audit 10）であり、本フェーズの対象外。

## 10. 総合判定と今後の実装判断基準

**総合判定: 現行の設計思想は本文中心・非代替・Wallace 裏方の 3 点で健全であり、違反文書はゼロ。欠けていたのは「本文へ帰着する」原則の明文化のみで、本フェーズで正典に固定した。**

完了条件の明文化（正典 §1 原則 1 に収載済み）:

> **StudyPanel is not a commentary.**
> **StudyPanel is not a Greek grammar teacher.**
> **StudyPanel is a reading support layer that helps users understand the biblical text and return to the text itself.**

今後の実装判断基準（Phase 24 以降のすべての設計・実装・検収に適用）:

1. **帰還テスト**: その説明を読み終えた読者は、本文のどこを見ればよいか判るか。判らなければ書き直す
2. **代替テスト**: その機能が完璧に動いたとき、読者は本文を読まなくなるか。なるなら設計が誤っている
3. **順序テスト**: 本文の観察 → 意味 → （必要なら）文法名称、の順になっているか。名称が先頭に来る文は不合格
4. **4 問テスト**: 新しい表示は 4 つの問い（何？/なぜこの訳？/どこまで一組？/前も見た？）のどれかに答えているか。どれにも答えない情報は追加しない
5. **静寂テスト**: 出せる内容がないとき、沈黙しているか（既存原則の再確認）
