## 1. 「読むための日本語」の定義

**読むための日本語(Reading Japanese)= ギリシャ語の形態・構造・意味を反映し、読者がギリシャ語本文を
読み進める助けとなる日本語**。ギリシャ語構造を保持し、隠さない。

| 対比 | 相手 | 違い |
|---|---|---|
| **翻訳との違い** | 翻訳 | 翻訳は目標言語として自然・完結した文を作る(語順再構成・意訳・構造の吸収)。Reading Japanese は**ギリシャ語構造を保持**し、読者がギリシャ語を追う「道しるべ」。翻訳は「日本語で完結」、Reading Japanese は「ギリシャ語への橋」 |
| **gloss との違い** | gloss(辞書的語義) | gloss は語単位・文脈非依存の訳語。Reading Japanese は**形態・構造・文脈を反映**した読める語形(彼→彼ら・格助詞・頭語 者/もの)。gloss より構造を持つが、翻訳ほど再構成しない |
| **既存日本語訳との違い** | 新改訳等の完成訳 | 完成訳は自然・語順自由・神学的整合を持つ翻訳文。Reading Japanese は**ギリシャ語順・逐語的だが破損しない**。完成訳は参照・比較用に保持(bible_data.japanese の出自) |
| **原文構造との関係** | ギリシャ語 | 形態(gender/number/case/person)・構造(関係詞 role/先行詞)・意味(語義)を日本語へ反映。**語順は原則ギリシャ語順を保持** |

## 2. japanese の最終責務

### bible_data 各章節データとして満たすべきこと

その章節のギリシャ語データ(全トークンの形態・構造・意味)に対し、**読者がギリシャ語構造を理解しながら
読める日本語**であること。具体的には:

- 明らかな破損がない(§4)。
- 誤解を招かない(ギリシャ語と異なる意味に読めない)。
- 章節内で一貫している。
- ギリシャ語の形態・構造が反映されている(隠されていない)。

### 語単位ではなく章節単位で評価する理由

- **最終成果物が「各章節の japanese」**であり、**読者は章節を読む**。
- 語単位で正しくても、章節として読むと**接続・一貫性・流れ**が崩れることがある(Stage F/G の知見)。
- したがって語単位の適切さは**必要条件**だが十分ではなく、**章節単位が主評価**(§8)。

## 3. ギリシャ語構造の反映原則

各層が日本語へ反映できる範囲。**根拠は既存データ(Macula / Louw-Nida / Abbott-Smith 注釈)にあるものに
限る。推論・創作はしない。**

| 層 | 反映する情報 | 反映の範囲 |
|---|---|---|
| **形態(Morph)** | gender / number / case / person / mood | 読む語形(彼/彼ら/それ・者/もの・私自身・格助詞・活用)。Morph v1 FROZEN の範囲 |
| **Syntax** | 関係詞 role / 先行詞リンク / 節構造 | 構造情報として反映(**語順は変えない**・K-5)。関係詞の役割・先行詞を読者が把握できる補助 |
| **Semantic** | 文脈依存の語義選択 / idiom / discourse | 既存注釈(lnGloss 等)に基づく語義の整理。**意味推論はしない** |

- 反映は**注釈に根拠があるもののみ**。省略語の生成・文脈からの意味断定はしない。

---

## 4. 「日本語としてふさわしい」の定義

「ふさわしい」は**自然さではなく**、以下の観点で定義する。

| 観点 | 定義 |
|---|---|
| **読解可能性** | 読者が意味を取れる(意味不明・破綻がない) |
| **構造忠実性** | ギリシャ語の形態・構造が日本語に反映されている(gender/number/role/先行詞 等が隠れていない) |
| **章節内の一貫性** | 同一章節内で語の扱いが一貫(同じ語が矛盾なく扱われる) |
| **語の接続** | 隣接語が日本語として接続する(助詞・活用が破綻しない) |
| **誤解を招く表現** | ギリシャ語と異なる意味に読める表現がない(例: 中性を「彼」と読ませる性数誤り) |
| **明らかな破損** | 二重助詞・不正活用・意味不明形がない |

- これらは「自然さ」ではなく **読める・忠実・一貫・接続・非誤解・非破損**。自然さは十分条件でも必要
  条件でもない(自然だが構造を隠す日本語は「ふさわしくない」)。

## 5. 「翻訳化」の境界

| 論点 | 定義 |
|---|---|
| **どこまでが Reading Japanese** | ギリシャ語構造を保持し、形態・構造・意味を反映した**逐語的だが読める**日本語。語順はギリシャ語順 |
| **どこからが翻訳** | 自然な日本語文のための**語順再構成・意訳・構造の吸収/隠蔽・意味補完**。ここからは翻訳=境界を越える |
| **日本語語順を変更してよい条件** | **原則変更しない**。Morph の語内整形(格助詞付与・活用)は語順変更ではない。真の語順再構成(関係節前置 等)は翻訳化であり**採らない**(K-5) |
| **ギリシャ語構造を隠してはいけないケース** | 形態(性数)・関係詞の役割/先行詞・語義の多義を、自然化のために消す/溶かすことは禁止(例: 彼ら→彼、関係節を自然文へ溶かす) |
| **意味の補完・推論の禁止範囲** | 既存注釈にない意味を補完・推論しない。文脈から意味を断定しない。省略語を生成しない |

## 6. Engine 責務境界

| 層 | 責務 |
|---|---|
| **Morph** | **何を決めるか**: 形態(gender/number/case/person/mood)から**読む語形**(語幹・格助詞・活用)を決定する。彼/彼ら/それ・者/もの・私自身 等。Morph v1 FROZEN |
| **Syntax** | **何を提供するか**: 構造情報(関係詞 role / 先行詞リンク / 節境界)を**提供**する。**語順変更・文字列生成はしない**(K-3/K-4/K-5) |
| **Semantic** | **何を整理するか**: 文脈依存の語義選択・idiom を**既存注釈に基づき整理**する。**推論はしない** |
| **Reading Japanese** | **何を最終決定するか**: Data 代表語を起点に、Morph/Syntax/Semantic の反映を統合し、**各章節の最終的な読む日本語を決定**する。破損なし・構造忠実・章節一貫を保証する最終責務 |

- 下位層(Morph/Syntax/Semantic)は**部分的な反映情報**を出し、Reading Japanese が**章節として統合・確定**する。

## 7. 改善・悪化の定義

japanese の before / after 比較の判定基準。

| 判定 | 定義 |
|---|---|
| **改善** | 構造忠実性の向上(性数/役割/先行詞が正しく反映)・**誤解の解消**(例: 彼→それ)・**破損の解消**・章節内一貫性の向上 |
| **中立** | 読解可能性・構造忠実性・一貫性が変わらない(自然さだけ変わった 等) |
| **悪化** | 構造忠実性の低下(構造を隠す)・**誤解の発生**・**破損の発生**・一貫性の低下・**既存注釈にない意味の混入(推論化)** |

- **「日本語がより自然になった」を改善の唯一の基準にしない。** 自然化の代償に構造を隠した/意味を
  推論で足した場合は、自然になっても **悪化**とする。
- 改善は「構造忠実・誤解解消・破損解消・一貫性」で測る。

## 8. 評価単位

| 単位 | 役割 |
|---|---|
| token | 局所の正しさ(必要条件の検査) |
| phrase | 語の接続・句内整合の検査 |
| clause | 節構造・役割の検査 |
| **verse(章節)** | **主評価単位**(最終成果物・読者が読む単位) |

- **主評価単位は verse(章節)**。理由: 最終成果物が「各章節の japanese」であり、読者は章節を読む。
- token/phrase/clause は**必要条件・中間検査**の単位。verse で最終評価する。

## 9. QA 方針

| 区分 | 内容 |
|---|---|
| **自動検査で判定できるもの** | 明らかな破損(二重助詞・不正活用・意味不明形)・構造忠実性の一部(性数反映の有無)・chip⇔panel 一致・Morph 回帰・Syntax 回帰・bible_data 差分監査・バイト等価(非破壊確認) |
| **章節単位の人間レビューが必要なもの** | 読解可能性・章節内一貫性・語の接続の自然さ・誤解を招くかの最終判断(Stage F/G 型の読者評価)。機械では確定しにくい |
| **Morph 回帰** | re-phase1 等の基準維持(morph 44,251・7 lemma) |
| **Syntax 回帰** | relativeSyntax 取得・K-3/K-4 基準(バイト等価・chip⇔panel 100%) |
| **bible_data 差分監査** | 章節の japanese 変更が意図した範囲のみであること。**重複ファイル(「 2.json」)混入の検出を含む**(H-5/H-6 の教訓)。実 FS・重複不在の確認 |

- QA は **自動(破損・回帰・差分)+ 章節人間レビュー(理解補助・一貫性)** の二段構え。
- 自動検査で「非破損・非回帰」を保証し、人間レビューで「章節としてふさわしいか」を評価する。
