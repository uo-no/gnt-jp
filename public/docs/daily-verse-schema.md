# DAILY_VERSES_365 スキーマ設計

**フェーズ**: D-1C  
**作成日**: 2026-07-15  
**種別**: データ構造設計書（実装なし）  
**参照**: `docs/daily-verse-year-plan.md`, `docs/daily-verse-slots.md`

---

## 概要

`DAILY_VERSES_365` は365件の日替わり聖句エントリを格納する配列。  
`getDailyVerse()` が UTC 日付ベースで決定論的にインデックスを選択する。

```
index = Math.floor(Date.UTC(y, m, d) / 86400000) % 365
```

配列インデックス 0 = 1月1日 とは限らない（UTC 通算日 mod 365）。  
インデックスと暦日の対応は `getDailyVerse()` 呼び出し時に動的に決まる。

---

## 必須フィールド

| フィールド | 型 | 制約 | 説明 |
|-----------|-----|------|------|
| `ref` | `string` | 非空 | 聖書箇所（書名 章:節 形式） |
| `lines` | `string[]` | 1件以上 | 画面表示行（`<br>` 区切りで結合） |
| `keywords` | `string[]` | 1件以上 | 節に実際に登場する語（対象 corpus の索引で 1件以上ヒット必須） |
| `corpus` | `'NT' \| 'LXX'` | enum | 検索対象コーパス。現在 `'NT'` のみ運用 |

**keywords 制約**:  
- 各語は `_exampleSearch(word)` 実行時に1件以上ヒットすること  
- 同一節から3〜5語程度（1語でも許容、6語以上は要検討）  
- 語幹・体言形を優先し、活用形は避ける（例: 「疲れた」→NG、「重荷」→OK）

### フィールド値形式

**`ref` — 書名 章:節**

```
書名（略号なし日本語） + 半角スペース + 章 + コロン + 節
```

```
OK:  'ヨハネ 14:6'
OK:  'コリント第一 13:13'
OK:  'ヨハネ第一 4:16'
OK:  'テサロニケ第一 5:16'
NG:  'ヨハネ14:6'          （スペースなし）
NG:  'Jn 14:6'             （英語略号）
NG:  'ヨハネ 14章6節'       （章節を漢字表記）
```

**`lines` — 改行単位の配列**

```
OK:  ['わたしは道であり、', '真理であり、命です。']
NG:  ['わたしは道であり、真理であり、命です。']  （1行に収めない）
```

文末句点（。）は最終行のみ。途中の行は読点（、）または体言止め。

**`corpus` — ASCII 大文字**

```
'NT'   新約聖書索引（現行）
'LXX'  LXX + 日本語索引（IDX.l3Ready 依存、未使用）
```

## theme 一覧

`theme` はフリーテキストだが `docs/daily-verse-slots.md` の各 Day の「テーマ：」を正典とする。

**形式**:

```
{上位テーマ}—{具体的焦点}
```

例:
```
新しい年の始まり—すべての初めにおられる神
信仰は望んでいることを保証する—ヘブル11:1
弱さの中の強さ—力は弱さのうちに完全に現れる
```

ダッシュ `—`（全角 EM ダッシュ）で上位概念と焦点を区切る。  
上位テーマ部分は当該週テーマと一致させること。

| 値 | 説明 | 主対応月 |
|---|------|--------|
| `'創造'` | 創世—神の設計 | 1月 |
| `'堕落'` | 罪の入り・神からの分離 | 3月 |
| `'契約'` | アブラハム契約・神の誠実さ | 2月、12月 |
| `'出エジプト'` | 解放の原型 | （散在） |

| `'律法の成就'` | 山上の垂訓・律法の深化 | 3月、7月 |
| `'律法と知恵'` | 知恵文学・召命・日常倫理 | 9月 |
| `'受難と贖い'` | 十字架・贖罪 | 4月 |
| `'復活'` | 死に勝つキリスト | 4月、11月 |
| `'新創造'` | 復活のいのちに生きる | 5月 |

## Season 一覧

| 値 | 期間 | 備考 |
|---|------|------|
| `'冬'` | 12月・1月・2月 | |
| `'春'` | 3月・4月・5月上旬 | |
| `'初夏'` | 5月中旬〜6月 | 梅雨前後 |
| `'夏'` | 7月・8月 | |
| `'秋'` | 9月・10月・11月 | |

月の中間で季節が変わる場合は移行節として両方を付与しない。  
配列インデックス（= 暦日）を基準に単一の値を選択する。

## バリデーション規則

実装時にチェックすべき条件（Phase D-2 完了後のリントスクリプト用）。

1. 配列長 = 365
2. 全エントリに `ref`, `lines`, `keywords`, `corpus` が存在する
3. `lines.length >= 1`
4. `keywords.length >= 1`
5. `corpus` が `'NT' | 'LXX'` のいずれか
6. `storyStage`（存在する場合）が定義済み enum 値のいずれか
7. `liturgicalTag`（存在する場合）が定義済み enum 値のいずれか
8. `season`（存在する場合）が定義済み enum 値のいずれか
9. 同一 `ref` の重複が ≤3 件（年間で同じ節が3回以上使われない）
10. 同一 `storyStage` が10日以上連続しない

## 現行 DAILY_VERSES_NT との差分

| 項目 | DAILY_VERSES_NT（現行） | DAILY_VERSES_365（予定） |
|------|----------------------|----------------------|
| 配列長 | 30件 | 365件 |
| 配列名 | `DAILY_VERSES_NT` | `DAILY_VERSES_365` |
| 取得関数 | `getDailyVerse()` | `getDailyVerse()`（変更なし） |
| corpus | `'NT'` のみ | `'NT' | 'LXX'`（LXX は将来） |
| theme | なし | 任意フィールドとして追加 |
| storyStage | なし | 任意フィールドとして追加 |
| liturgicalTag | なし | 任意フィールドとして追加 |
| season | なし | 任意フィールドとして追加 |

**移行方針（Phase D-4 実施時）**:  
1. `DAILY_VERSES_365` 配列を別ファイルまたは同ファイル末尾に追加  
2. `getDailyVerse()` の参照先を差し替え  
3. `DAILY_VERSES_NT` を削除  
4. `mod` 値を 30 → 365 へ変更（`getDailyVerse()` 内）
