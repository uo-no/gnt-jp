# 関係詞 Syntax Rule 責務設計(Stage K-1)

策定日: 2026-07-20
位置づけ: **Morph Rule Engine v1 FROZEN 後の最初の Syntax Layer 設計**。関係詞の自然な日本語化に
必要な Syntax の責務を定義する。J-6a/J-6b で「関係詞の case=節内役割=Syntax」と確定した内容を、
実データ(Macula 注釈)に基づいて具体化する。
根拠: docs/relative-morph-rule-design.md(J-6a)・relative-morph-rule-implementation-report.md(J-6b)・
morph-rule-engine-v1-frozen.md(J-9)。

**本文書は設計(責務定義)のみ。実装・コード・擬似コード・修正案は含まない。**
reading-engine.js / bible_data / Semantic / Presentation は変更しない。

数値は **クリーン NT 実測**(総 137,741・重複「 2.json」不在)。対象: ὅς(G3739)/ ὅστις(G3748)/
ὅσος(G3745)、関係詞 総数 1,658。

---

## 1. 関係詞の case が節内役割を持つことの確認

関係詞は **先行詞と gender/number で一致**し、**case は関係節内での文法役割**を表す(Greek 文法)。
これを実データ(case × 注釈 `role` フィールド)で確認した。

| case | 節内 role の分布(実測) | 主たる役割 |
|---|---|---|
| **nominative** | s(主語)520 / p 4 / - 9 | **節の主語**(who) |
| **accusative** | o(目的語)521 / s 22 / adv 21 / - 92 | **節の目的語**(whom/which) |
| **dative** | io(間接目的)38 / adv 53 / o 31 / - 94 | 間接目的・前置詞句(to whom) |
| **genitive** | adv 68 / o 51 / p 7 / - 115 | 所有・前置詞句(whose / of which) |

- **nominative→主語・accusative→目的語 が強い対応**(それぞれ 520/534・521/639)。**case が節内役割を
  担う**ことがデータで裏付けられた。
- dative/genitive は case だけでは役割が一意でない(間接目的・副詞的・所有が混在)→ 後述の `role`
  注釈で補完される。

---

## 2. 先行詞との関係

**決定的所見: 先行詞が `referent` フィールドに注釈済み**(Macula 由来)。

- 関係詞 1,658 件のうち **`referent` 充足 1,071 件(64.6%)** が先行詞トークン ID を保持。
- **`referent` 無し 587 件(35.4%)** は先行詞を持たない **自由関係(free relative)**候補
  (whoever / whatever)。
- Greek の一致規則: 関係詞の **gender/number は先行詞に一致**(→ 頭語の種別=Morph が担当済)、
  **case は節内役割**(→ Syntax が担当)。

| 先行詞 | 件数 | 種別 | 責務 |
|---|---|---|---|
| referent 有 | 1,071 | 束縛関係(先行詞あり) | Syntax(先行詞連結・節配置) |
| referent 無 | 587 | 自由関係(whoever/whatever) | Semantic(自由関係の語義) |

---

## 3. 主語 / 目的語 / 所有関係の分類

**`role` フィールドが節内役割を明示注釈**している(全関係詞に付与)。

| role | 意味 | 件数 | 日本語での節内標示(方向性) |
|---|---|---|---|
| **s** | 主語 | 543 | 〜が/は(節の主語) |
| **o** | 目的語 | 603 | 〜を(節の目的語) |
| **adv** | 副詞的 | 143 | 〜に/で 等(前置詞句・状況) |
| **io** | 間接目的 | 38 | 〜に(与える相手 等) |
| **p** | 述語 | 13 | 〜である(述語関係) |
| **o2** | 第二目的 | 8 | 二重目的 |
| **-** | 未標示 | 310 | (role 注釈なし・case で推定) |

- **role は case とほぼ対応**(nom=s / acc=o)しつつ、case では曖昧な dative/genitive の役割
  (io / adv / 所有)を補完する。
- **所有関係(whose)は genitive + role により識別**(genitive の o/adv/所有的用法)。

## 4. Morph 責務との境界確認

| 情報 | 担当層 | 内容 |
|---|---|---|
| gender → 頭語(者/もの) | **Morph(実装済 J-6b)** | 先行詞の性に一致する referent 種別(人=者 / 事物=もの) |
| number | Morph(頭語は単複兼用) | — |
| **case → 節内役割** | **Syntax(本設計)** | 主語/目的語/間接/所有。頭語への機械的助詞付与では表せない |
| **先行詞連結** | **Syntax(本設計)** | referent が指す先行名詞との連結・節の配置 |
| 節の動詞・修飾内容 | Semantic/Presentation | 「〜する」に相当する節内容の実体化 |

- **Morph は頭語の種別選択に閉じる**(J-6b で確定)。**case・role・referent は Syntax**。
- 決定的差: αὐτós/τίς では case→格助詞が Morph だったが、**関係詞の case は節内役割(Syntax)**であり、
  Morph が格助詞を付けることは v1 で明示的に対象外とした(J-6b)。本設計はその Syntax 側を定義する。

---

## 5. Syntax で解決すべき範囲

| 課題 | 入力(注釈) | Syntax の責務 |
|---|---|---|
| **節内役割の標示** | `role`(s/o/io/adv/p)・case | 関係節内で頭語(者/もの)に対応する日本語の役割標示(主語/目的語/間接/所有)を決める |
| **先行詞連結** | `referent` | 関係詞が指す先行名詞を同定し、日本語の**連体修飾(関係節が先行詞の前に来る)**構造へ連結 |
| **節境界の確定** | トークン列・role | 関係節に属するトークン範囲を確定(どこからどこまでが「〜する」節か) |
| **語順再構成** | 上記 | Greek(先行詞 + 関係節)→ 日本語(関係節 + 先行詞)の順序反転 |

- 中核は **`role` と `referent` の注釈を読み取り**、関係節の役割標示と先行詞連結を行うこと。
  **これは推論(parse)ではなく注釈の読み取り**であり、Morph が gender/number を読むのと同型
  (プロジェクト原則「意味を推論しない・既存注釈を使う」に合致)。
- **語順再構成(節を先行詞の前へ)** は Presentation/句レンダリング(PhraseRenderer)との協働が要る
  より複雑な部分。

---

## 6. Semantic へ委譲すべき範囲

| 用法 | 対象 | 内容 |
|---|---|---|
| **自由関係(whoever/whatever)** | referent 無 587 件 | 先行詞なし。「〜する者は誰でも」等の任意性は語義 |
| 不定関係 | ὅστις | whoever・総称(they)の含意 |
| 数量 | ὅσος | as many/how much(J-6b で頭語化・数量は Semantic) |
| 節の動詞の語義 | 全般 | 「〜する」の実体(動詞の意味)は動詞側の責務 |
| discourse | 全般 | 先行詞の既出性・指示対象の精緻化 |

## 7. Rule Engine 化可能性

| 観点 | 評価 | 根拠 |
|---|---|---|
| 節内役割の取得 | **高**(決定的) | `role`(s/o/io/adv/p)が**全関係詞に注釈済**。case との対応も強い |
| 先行詞の取得 | **中〜高** | `referent` が **64.6% 充足**。残 35% は自由関係(Semantic 側で識別可能) |
| 役割→日本語標示 | **高** | role→役割助詞(が/を/に/の)は決定的マッピングが可能 |
| 節境界の確定 | **中** | role 分布とトークン列から推定可能だが、複文・入れ子で難度上昇 |
| 語順再構成(節→先行詞前) | **低〜中** | 日本語連体修飾への並べ替えは PhraseRenderer 級の再構成が必要 |

- **`role`/`referent` を読み取る Rule Engine は成立可能**(Morph Registry と同型の「注釈読み取り」方式)。
  推論を要さないため、決定的・凍結可能な設計にできる。
- ただし **「自然な関係節の日本語」= 語順再構成**は、役割標示より一段複雑で、レンダリング層との
  協働を要する。Syntax Rule Engine の**第一段は役割標示・先行詞連結の情報付与**、**第二段が語順再構成**
  という段階化が妥当。

---

## 8. 実装 Gate 判定

| Gate | 状態 | 理由 |
|---|---|---|
| G-data(role) | **充足** | `role` 全関係詞注釈済(s/o/io/adv/p) |
| G-data(referent) | **部分充足** | 64.6%。残は自由関係=Semantic で識別 |
| G-deterministic(役割標示) | **充足** | role→役割助詞は決定的・推論不要 |
| G-render(語順再構成) | **未充足** | 節→先行詞前の並べ替えは PhraseRenderer 協働が必要・未整備 |
| G-boundary(節境界) | **条件付** | 単純節は可・複文/入れ子は難 |

### 判定

- **Syntax Rule Engine 化は「役割標示・先行詞連結」の範囲で成立可能**(role/referent 注釈を読み取る
  決定的方式)。これは Morph v1 の Registry 方式を Syntax へ拡張する自然な発展である。
- **ただし「自然な日本語の関係節」= 語順再構成は本設計の Gate では未充足**(PhraseRenderer 級の
  再構成・節境界の確定が前提)。したがって **K-2 以降の実装は段階化**する:
  1. **第一段(実装可能候補)**: `role`→節内役割標示、`referent`→先行詞連結の**情報付与**
     (頭語 者/もの に役割助詞を与える等・推論なし)。
  2. **第二段(要 PhraseRenderer)**: 節→先行詞前の**語順再構成**による自然な連体修飾化。
- 自由関係(referent 無 587)・数量(ὅσος)・不定(ὅστις)は Semantic へ委譲(本設計で確定)。
- **本設計は責務定義のみ。実装は行わない。**
