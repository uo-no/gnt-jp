## 1. 全体品質サマリー

- **Data 層は健全**: placeholder/空 japanese **0**・重複 tokenId **0**・ユニーク 137,741(既知 NT 語数と一致)。
  過去の誤データ(μόδιος・σύ G5213)は H-2/H-5 で修正済。**Data 品質は高い**。
- **Morph 層はほぼ完成**: Morph Rule Engine v1(7 lemma・morph 44,251)が gender/number/person の
  決定的ケースを処理済。**形態で決まる読む語形は概ね反映されている**。
- **Syntax 層は情報保持まで**: 関係詞の role/referent は K-3/K-4 で取得・保持(バイト等価)。ただし
  **構造は読む日本語へ未反映**(StudyPanel Integration は設計段階 K-5)。指示詞の連体/代名詞判定も未実装。
- **Semantic 層はほぼ未実装**: 文脈依存の語義・intensive・person-leveling・多義語は一部 lnGloss を除き未処理。
- **Reading Japanese(章節)**: 直近の章節評価(quality-v1・200 節サンプル)は A130/B55/C15。ただしこれは
  Morph v1 実装**前**の基準で、以後 2,678 トークン改善済(章節単位の再サンプルは未実施)。

**総括**: **Data・Morph は完成度が高く、残る主要ギャップは Syntax(構造反映)と Semantic(語義選択)**。
現状の japanese は「Data 代表語 + Morph 反映」までは良好で、構造・文脈の反映が次の課題。

## 3. 問題分類(問題内容・原因・代表例。修正案なし)

### 3-1. Data 起因

- **問題内容**: 明らかな誤データ・固定 gloss 残存・欠落・重複・破損。
- **原因**: 元データの誤り・同期コンフリクト。
- **現状**: **ほぼ解消**。placeholder 0・重複 0。過去例 μόδιος(姦淫の女→枡)・σύ G5213(彼→あなたがた)は
  修正済。代表的残存リスクは §6(同期重複)。

### 3-2. Morph 起因

- **問題内容**: 性・数・人称が固定 japanese に反映されない。
- **原因**: 単一固定値が形態変化を表現できない。
- **現状**: **v1 で解消**(彼→彼ら/それ・誰→何・者→もの・自分自身→私自身 等)。
- **代表例(解消済)**: αὐτós 性数・τίς 性・関係詞頭語・ἐμαυτοῦ/σεαυτοῦ person。

### 3-3. Syntax 起因(**残ギャップ最大の一つ**)

- **問題内容**: 節構造・修飾関係・主語/目的語関係が読む日本語に反映されない。
- **原因**: 構造情報(関係詞 role/先行詞・指示詞の連体/代名詞)が japanese に統合されていない。
- **代表例**:
  - **関係詞 1,658**: 「〜する者/もの/だけ」が placeholder のまま(節の役割・先行詞リンクは K-3/K-4 で
    取得済だが未反映)。
  - **指示詞 1,687**: οὗτος「この」/ ἐκεῖνος「あの」/ τοιοῦτος「このような」が連体/代名詞の区別なし
    (J-7a で Syntax 依存と確定)。

### 3-4. Semantic 起因

- **問題内容**: 同一形態でも文脈・語義で読みが変わる。
- **原因**: 語義選択・discourse・person-leveling が未処理。
- **代表例**:
  - **ἑαυτοῦ 823**: person(私たち自身/彼ら自身)が節主語依存(J-8a)。
  - **τις 525**: some/any(ある/誰か/何か)の不定。
  - **αὐτós intensive 約 97 / τίς why-which 約 106**: 形態同一で語義が異なる残(Morph 済の内側)。
  - **多義語**(κόσμος 世界/世・λόγος ことば/申し開き 等・H-3c): lnGloss で一部処理・残は文脈選択。

### 3-5. Reading Japanese 起因(章節)

- **問題内容**: 章節として読む日本語の品質(読解困難・語の接続・構造忠実性・一貫性・誤解・構造隠蔽)。
- **原因**: 語単位で正しくても章節の流れ・接続で不足が生じる(L-0 §2)。
- **代表例**: quality-v1(Stage G・200 節サンプル)の C 判定 15 件(語順断片・命令形・疑問詞助詞 等)。
  ※サンプル基準・Morph v1 前。

## 4. 優先順位(改善効果が高い順)

| 順 | カテゴリ | 理由 |
|---|---|---|
| **1** | **Syntax(関係詞 1,658)** | 情報は K-3/K-4 で取得済・L-0 の「読むための Syntax」方針(K-5)で反映経路が設計済。最も着手しやすく効果が構造忠実性に直結 |
| **2** | **Semantic(ἑαυτοῦ 823 / 不定 525 / 多義語)** | 件数大・誤解に直結(person・語義)。ただし文脈判断が要り難度は中〜高 |

| **4** | **Reading Japanese(章節)** | 章節単位の一貫性・接続。人間レビュー主体で漸進 |
| — | Data / Morph | **完成度が高く優先度低**(維持・監視) |

## 5. L-2 Reading Japanese Builder への入力(責務切り分け)

| 責務 | 内容 |
|---|---|
| **Builder が解決すべき問題** | Data 代表語 + Morph/Syntax/Semantic の反映を**章節として統合**し最終的な読む日本語を確定(L-0 §6)。破損なし・構造忠実・章節一貫の保証 |
| **Morph へ戻すべき問題** | **ほぼ無し**(v1 完成)。真の Morph 対象漏れが将来出れば Registry 登録 |
| **Syntax 拡張が必要な問題** | 関係詞の役割/先行詞の**読解反映**(K-5 StudyPanel Integration)・指示詞の連体/代名詞判定 |
| **Semantic が必要な問題** | ἑαυτοῦ person・τις 不定・αὐτós intensive・τίς why/which・多義語の文脈選択 |

- **Builder は「翻訳」ではなく統合**(L-0)。各層の反映情報を章節として組み立て、構造を隠さず確定する。

## 6. リスク(現状のまま残すと今後問題になるもの)

| リスク | 内容 |
|---|---|
| **同期重複(「 2.json」)の再発** | クラウド同期下で一括 write 時にコンフリクトコピーが生成され全監査を水増しする(H-5/H-6 で実害・修正済)。**bible_data 一括変更後は必ず重複検出する**運用が必須 |
| **章節評価の陳腐化** | quality-v1 は Morph v1 前の 200 節サンプル。現状を反映した章節単位の再評価が未実施 |
| **Syntax 未反映の固定化** | 関係詞「〜する者」placeholder が長期放置されると、後続の Semantic/章節評価が誤った前提に立つ |
| **Semantic の誤実装リスク** | 文脈語義は推論に傾きやすい。L-0 の「意味推論禁止・既存注釈のみ」を破ると構造隠蔽=悪化を招く |
| **改善基準の取り違え** | 「自然になった」を改善と誤認すると構造忠実性を損なう(L-0 §7) |

| 層 | 完成度(推定) | 根拠 |
|---|---|---|
| **Morph** | **約 90%** | v1 で決定的な gender/number/person を処理・7 lemma・残は非 Morph。真の対象漏れ僅少 |
| **Syntax** | **約 25%** | Phase 1/2(情報取得・保持・バイト等価)完了。読解反映(StudyPanel Integration)は設計段階・未実装 |
| **Semantic** | **約 15%** | lnGloss(Phase 5)で一部の語義整理のみ。person/intensive/不定/多義語は未処理 |
| **Reading Japanese** | **約 55〜60%** | quality-v1 で A65%(読める)だが Morph v1 前・構造/語義ギャップ残・章節再評価未実施 |
| **bible_data.japanese** | **約 90%(Data 代表語として)** | 欠落 0・重複 0・誤データ解消・安定代表語。ただし最終 Reading Japanese としては上位層反映に依存 |

### プロジェクト全体の完成度(推定)

**約 50%**。

- 内訳の考え方: 最終目的は「各章節の読む日本語」。Data・Morph(基盤)は概ね完成(~90%)だが、
  最終品質を左右する **Syntax(~25%)と Semantic(~15%)が未整備**で、章節としての読む日本語は
  半ばに留まる。
- **基盤(Data/Morph)は堅く、残作業は上位層(Syntax 反映・Semantic 語義・章節統合)に集中**する段階。
