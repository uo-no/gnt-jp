## 1. 現在の検索フロー

### 1a. 初回ロード

```
ページ表示
 │
 ├── [左] 検索パネル
 │     ├── 聖書原文検索（ヘッダー）
 │     ├── 準備中… / NT検索できます → 消える（idx-pill）
 │     ├── 検索語ラベル + 入力欄（Gentium フォント・日本語サジェスト対応）
 │     ├── 検索するボタン
 │     ├── 語根で探す（常時ON）
 │     └── › 詳細検索（collapsed）
 │
 └── [右] 結果エリア
       ├── λόγος グリフ（大・薄）
       ├── 聖書を検索する
       ├── 日本語・ギリシャ語・Strong番号で検索できます
       ├── 例: 愛 / 信じる / ἀγαπάω / 罪 / 希望 / 恵み（exチップ）
       └── テーマから探す → パターンチップ（4件・非同期表示）
```

### 1b. 日本語検索（例: 愛）

```
入力 → [Concept Insight 発火チェック]
         ├── 登録語: resolveConcept → Term[] → lemma検索 AND
         │   Insight カード表示（ギリシャ語語彙 / 関連概念 / さらに詳しく）
         └── 未登録語: searchJapanese のみ
         
結果エリア:
  Concept Insight（愛: ἀγαπάω・ἀγάπη … + 関連概念 + 詳しく）
  ↓
  N件（M lemma）
  ↓
  Lemmaグループ一覧（ἀγαπάω 353件 ▶ クリックで展開 …）
    └── 展開後:
          特徴共起（共起チップ）
          代表例（Hit Card × 3 + さらに表示）
            └── Hit Card:
                  参照（ヨハネ3章16節）   [語根]
                  「神はその独り子を…」（訳文・非同期ロード）
                  [原文を見る][本文で見る]
                  └── 原文展開: FLOWチップ（節全体の語義）
  ↓
  出現分布（末尾・書物別バーチャート・クリックで絞り込み）
```

### 1c. ギリシャ語検索（例: ἀγαπάω）

```
入力 → detectQueryType = 'greek' → searchGreek
         （Concept Insight 発火なし）

結果エリア:
  N件 / M節
  ↓
  Hit Card × 3（最初）
  [残り N 節を表示]（折りたたみ）
  Hit Card × 77（折りたたまれている）
  ↓
  出現分布（末尾）
```

### 1d. 高度機能フロー

```
詳細検索を開く
 ├── フレーズで探す: 2語以上でフレーズ一致検索（日本語: 1語でも動作）
 ├── 近くに出てくる語: ±N語（スライダー＋プリセット）+ 共起パネル
 └── 品詞・語形で絞る: POS/格/数/性/時制/態/法 チップ

パターン検索（空き状態のみ表示）:
 互いに＋命令形 / μή＋命令形 / ἵνα＋接続法 / ἐάν＋接続法
 → runTermProximitySearch（内部: resolveTerm × 2 + verse scope）
```

## 2. 良い点

| 機能 | 評価理由 |
|---|---|
| 単一入力ボックス | 日本語・ギリシャ語・Strong番号を1つの欄で処理。入力タイプ自動判定 |
| Concept Insight | 「愛を検索 → 関連するギリシャ語 → 辞書」の流れが自然。表示専用 |
| Progressive Disclosure | 詳細検索が `<details>` に格納。初心者を迷わせない |
| 翻訳の自動ロード | 日本語訳を async で先行表示。ユーザーが触れる前から読める |
| 「本文で見る」ボタン | Hit Cardの最右に配置。青色で目立つ。正典順URLリンク |
| 近接検索の可視化 | prox-dist-row で2語間の距離を視覚表示 |
| 書物分布（末尾） | UX-2で移動済み。補足情報として適切な位置 |
| モバイルドロワー | 680px以下でBottom Drawer + FAB。タッチ最適 |
| URL共有 | syncURL() で検索状態をURLに反映。ブックマーク可能 |

## 3. UX問題一覧

### カテゴリ別

#### 3.1 初回利用体験

| 項目 | 重要度 | 問題 | 改善方向 |
|---|---|---|---|
| ページタイトル | ★★★☆☆ | `<title>検索ツール</title>` — 機能説明として弱い | 「聖書原語検索 — 聖書原語リーダー」等 |
| Strong番号の可視性 | ★★★☆☆ | `empty-desc` テキスト内に書かれているが小さい。例チップに G25 等がない | 例チップに「G25」を1件追加 |
| Concept 展開の非可視性 | ★★★☆☆ | ユーザーは「愛」を入力したとき Concept と直接 Japanese 検索の違いを理解できない | 差別化不要（内部実装）。Insight の存在自体で十分 |
| 「準備中」インジケータ | ★★★☆☆ | 初回ロード時に "○ 準備中…" が左上に表示され続ける（約6秒）。不安感 | 高さ固定・消去アニメは適切。メッセージをもう少しポジティブに（例:「データを読み込んでいます」） |
| 例チップの質 | ★★★☆☆ | 愛・信じる・罪など全て日本語概念。ギリシャ語・Strong番号の使い方が想像しにくい | ギリシャ語例（ἀγαπάω）と Strong 例（G25）を追加 |

#### 3.2 検索入口の整理

| 項目 | 重要度 | 問題 | 改善方向 |
|---|---|---|---|
| パターン検索の消滅 | ★★★★☆ | 4つのパターンチップが**空き状態のみ**表示。検索後は消える。再発見できない | サイドバーの履歴ブロック直前または下に常駐セクションとして配置 |
| サジェストがJapaneseのみ | ★★★☆☆ | ギリシャ語入力時はドロップダウンが出ない。タイプ中のフィードバックゼロ | 近似lemmaのサジェストは実装難度高。最低限「該当なしの場合なし」の状態を維持 |
| 検索語ラベル `s-label` | ★★☆☆☆ | 「検索語」というラベルが入力欄の上にある。入力欄自体にプレースホルダーがあれば不要 | 削除候補（入力欄のみで自明） |

#### 3.3 検索入力体験

| 項目 | 重要度 | 問題 | 改善方向 |
|---|---|---|---|
| フレーズ検索のサイレント失敗 | ★★★★☆ | 「フレーズで探す」ONで1語入力 → phrase layer は `words.length >= 2` でないと実行されない → 結果は lemma のみ（エラーなし・説明なし） | phrase 有効・1語入力時に phrase-sub の注記を更新「2語以上入力してください」 |
| 空検索のフォーカスのみ | ★★★☆☆ | 空入力で検索ボタン → 入力欄をfocusするだけ。メッセージなし | バリデーションメッセージ（例: プレースホルダーを赤に変えてflash） |
| ギリシャ語入力の補助 | ★★★☆☆ | ギリシャ語キーボードなし環境でのギリシャ語入力は困難。フォールバックなし | アクセント除去検索は実装済み（normalizeGreek）。案内文言の追加のみで可 |
| プレースホルダーの情報量 | ★★★☆☆ | `愛・信じる・ἀγαπάω …` — 「…」で省略されているが Strong番号や概念について示せていない | `愛 / ἀγαπάω / G26 …` に変更 |

#### 3.4 検索結果体験

| 項目 | 重要度 | 問題 | 改善方向 |
|---|---|---|---|
| 初期表示3件 (`FOLD = 3`) | ★★★★★ | ギリシャ語検索で最初に見えるのは3件のみ。残りは折りたたみ。「これだけか」という印象 | `FOLD` を 5〜8 に増やす |
| Lemmaグループの2段階展開 | ★★★★☆ | 日本語検索 → グループ一覧 → クリック展開 → hit card の2ステップ。聖句を読む前に2アクション必要 | グループの先頭1件を常時展開表示、もしくは最大件数グループを自動展開 |
| グループ展開の Affordance | ★★★★☆ | `lg-toggle` の ▶ チェブロン（9px）が小さすぎる。「クリックで展開」が分からない | lg-bar 全体がクリック可能であることを hover スタイルで強調（現在も `.lg-bar:hover` はあるが視覚的に弱い） |
| 「訳が見つかりません」 | ★★★☆☆ | LXXのhit cardで翻訳が取得できない場合に表示される。OT検索では頻出 | LXXである場合は「旧約（ギリシャ語版）」と表示し、翻訳なしを想定内にする |
| hit-count の表記揺れ | ★★★☆☆ | Greek: "N 節" / Japanese: "N件（M lemma）" — 単位が異なる | 統一（"N節 / M語根" など） |
| 「特徴共起」ラベル | ★★★☆☆ | Lemmaグループ内の co-occurrence セクション名。神学的でなく技術的 | 「一緒に使われる語」「よく一緒に出る語」 |
| CI「さらに詳しく」の発見性 | ★★★☆☆ | Concept Insight の `<details>` が閉じているため、pattern/syntax への導線が見えない | デフォルトで関連する概念は展開しておく（patternとsyntaxだけ折りたたみ） |

#### 3.5 検索後の学習導線

| 項目 | 重要度 | 問題 | 改善方向 |
|---|---|---|---|
| 「本文で見る」後の戻り導線 | ★★★★☆ | `本文で見る` をクリックすると `index.html` に遷移。ブラウザバックのみで戻れる | `target="_blank"` で別タブ開きに変更、または「← 検索に戻る」を index.html 側に追加 |
| PatternSearch 後の Insight なし | ★★★☆☆ | `runPatternSearch()` 実行後は `_activeInsightConcept = null` で CI が消える。文脈が断絶する | Pattern 実行時もそのパターンの説明を CI 領域に表示（description フィールドを活用） |
| Wallace文法への導線の深さ | ★★★☆☆ | Wallace 構文パターン → CI「さらに詳しく」の中の「関連する文法」チップ → 2段階 | CI「関連する文法」を常時表示（さらに詳しくの外に出す）、概念に syntaxTypes がある場合のみ |
| FLOWチップの役割の不明さ | ★★☆☆☆ | 「原文を見る」展開後に並ぶ節全体の語義チップ。何をするものか説明がない | ツールチップ or ラベル「節の全語一覧」 |

#### 3.6 履歴機能

| 項目 | 重要度 | 問題 | 改善方向 |
|---|---|---|---|
| セッション限定 | ★★★★☆ | `_history` はメモリ内配列。リロードで全消去 | `localStorage` に保存（JSON文字列で保存・復元） |
| 削除機能なし | ★★★☆☆ | 個別削除・一括クリアともにない。最大8件でローテーション | ×ボタン（削除）を各履歴アイテムに追加 |
| Layer badge が英語 | ★★★☆☆ | `lemma / phrase / prox / morph` の英語コードが小バッジで表示。内部用語 | 「語根」「フレーズ」「近接」「品詞」に変更（BADGE_JP と同じ対応表を使用） |
| 表示位置 | ★★★☆☆ | 履歴ブロックが詳細検索の下に配置。スクロールしないと見えないことがある | 検索ボタン直下か、サイドバー最下部に固定 |

#### 3.7 高度機能

| 項目 | 重要度 | 問題 | 改善方向 |
|---|---|---|---|
| パターン検索の再アクセス | ★★★★☆ | 空き状態でしか見えない（検索後は消える）。再実行のためには検索語を消す必要あり | → 3.2「パターン検索の消滅」と同一。サイドバー常駐が解決 |
| フレーズ検索のヒント | ★★★☆☆ | `phrase-sub` には「複数語を空白で区切って」の説明があるが、layer ON→1語の失敗時に表示されない | `unifiedSearch` の冒頭でフレーズON・1語入力時に `phrase-sub` をハイライト |
| 形態論チップのコード表示 | ★★★☆☆ | morph-chip は `N 名詞`、`V 動詞` のように英語コード先行。視覚的にコードが目立つ | 日本語先行（`名詞 N`）に変更、もしくは日本語のみでツールチップにコード |
| 品詞で絞るのみで検索できる | ★★☆☆☆ | morphのみON（lemma/phrase/prox OFF）で検索すると `レイヤーを選択してください` エラー | morph単独での検索を可能に（全トークンを morph でフィルタ）か、UIでmorph単独を禁止する旨を明示 |

## 4. 削除候補

| UI要素 | 根拠 | 現状 |
|---|---|---|
| `s-label`「検索語」 | 入力欄のプレースホルダーと役割重複。占有スペースに対して情報価値ゼロ | 削除推奨 |
| `cooc-col`（共起パネル・近接検索時のみ） | 結果エリアの右カラムに出現するが、大半のユーザーは使い方を理解しない。近接検索ユーザーのみ有用 | 削除ではなく、近接検索時の「よく一緒に出る語」セクションをヒットリストの上部に移動 |

## 5. 優先改善案

### ★★★★★ 最優先（利用者の中断を防ぐ）

| # | 改善 | 対象 | 理由 |
|---|---|---|---|
| P1 | `FOLD` を 3 → 8 に増やす | `_renderHitCards()` L3316 | 3件のみでは「結果が少ない」印象を与え、離脱につながる |
| P2 | 最大件数のLemmaグループを自動展開 | `_renderLemmaGroups()` | 日本語検索で聖句を読むまでに2ステップ必要。1ステップに削減 |

### ★★★★☆ 高優先（発見性・継続利用に影響）

| # | 改善 | 対象 | 理由 |
|---|---|---|---|
| P3 | パターンチップをサイドバーに常駐 | HTML + CSS | 検索後に消えるため再発見不可。4件だけなのでスペースは少ない |
| P4 | 履歴を `localStorage` に永続化 | `_addHistory()` / `_renderHistory()` | セッション破棄で消えると「前回の検索」が無価値 |
| P5 | `本文で見る` を `target="_blank"` に | `_buildUnifiedCard()` L3501 | 検索コンテキストを失わずに聖書本文を開ける |
| P6 | フレーズ検索・1語時の案内 | `unifiedSearch()` + `phrase-sub` | サイレント失敗で「フレーズ検索は動かない」と誤解される |

### ★★★☆☆ 中優先（情報設計の精度）

| # | 改善 | 対象 | 理由 |
|---|---|---|---|
| P7 | 履歴バッジを日本語化（語根/フレーズ/近接/品詞） | `_renderHistory()` | 内部コードをユーザーに見せない |
| P8 | CI「関連する文法」を常時展開（syntaxTypes>0 の概念のみ） | `_renderConceptInsight()` | Wallace文法への導線が「さらに詳しく」の中に埋もれている |
| P9 | 例チップに G25 を追加 | HTML empty state | Strong番号の使い方を具体的に示す |
| P10 | 「特徴共起」→「よく一緒に出る語」 | `_renderLemmaGroups()` L3632 | 専門用語をユーザー語に変換 |
| P11 | Pattern 実行時に CI 領域にパターン説明を表示 | `runPatternSearch()` | 文法パターンの意味が分からないまま結果を見ることになる |
