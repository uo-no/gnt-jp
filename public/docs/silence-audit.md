# UX-11-5 — Silence Audit

**実施日**: 2026-07-15  
**種別**: 監査のみ（実装変更なし）  
**対象**: 画面内の「静寂」と「ノイズ」の分布評価

---

## 「余白」と「静寂」の違い

**余白（Whitespace）**: 要素と要素の間の空間。  
**静寂（Silence）**: その空間が何かを語っている状態。

Apple が追求しているのは「余白を置く」ことではなく「静寂を作る」こと。

| 比較 | 余白 | 静寂 |
|------|------|------|
| 目的 | 要素を分離する | コンテンツに「間」を与える |
| 効果 | 圧迫感の軽減 | 「ここに意味がある」というシグナル |
| 感触 | 息継ぎ | 沈黙の中の問いかけ |
| 失敗例 | コンテンツが足りないだけの空白 | 存在しない（意図的なら常に静寂）|

美術館で絵画を1枚だけ壁に飾る——その周囲の白壁は「余白」ではなく「静寂」。  
その静寂が絵画の価値を高める。

---

## 評価指標

| 指標 | 説明 | スケール |
|------|------|--------|
| Silence | 意図的な静寂の量と質 | ★〜★★★★★ |
| Noise | 視覚的・情報的ノイズの量 | ★〜★★★★★（多いほど悪い）|
| Signal Clarity | 本当に伝えたいものが静寂の中で際立つか | ★〜★★★★★ |

### 右パネルの静寂評価

**「今日は、どんなことばを読みますか？」の周囲の空白**

→ これは**静寂に近い**。テーマ語に「間」を与えている。問いかけとテーマ語の間に沈黙がある。

**テーマ語「証し」以下の空白**

→ これは**「足りないだけの空白」に近い**。画面の下半分（約40%）がただ白い。  
「意図的な間」というより「コンテンツが終わった後の空白」に見える。

**理由**:  
意図的な静寂は「そこに何かある予感」を残す。  
現状の下部空白は「それだけ」で終わっている。テーマ語の縦間隔を広げ、下部余白を「リズムの延長」にすることで静寂に変わる。

---

### 左パネルの静寂評価

**左パネルに静寂はない。**

| 要素 | 静寂への貢献 |
|------|------------|
| ヘッダー区切り線 | 分離はするが、静寂ではない |
| 入力欄の下余白（10px）| 余白。静寂ではない |
| 検索ボタン（高密度） | ノイズ |
| 語根カード（高密度） | ノイズ |
| 詳細検索下の余白 | 余白。静寂ではない |

左パネルはすべての余白が「機能的なスペーシング」として機能しており、「静寂」がない。  
これが左パネル全体を「道具箱」に見せる原因。

## 4. ノイズの種類と分類

### Type 1: 点ノイズ（Point Noise）

単発だが重量が高い要素。  
例: `run-btn`、Mobile FAB

影響範囲: 局所的だが視線を強く引き寄せる。

### Type 2: 面ノイズ（Area Noise）

塗りやカードが面積を持つ要素。  
例: `.layer-row.active-lemma`（カード面積全体が青ティント）、Concept 概念チップ（青塗り4個）

影響範囲: エリア全体の密度を上げる。

### Type 3: 繰り返しノイズ（Repetition Noise）

同一スタイルの要素が N 回繰り返される。  
例: `hit-open-btn-lg`「本文で読む」× 631件、`ht-badge`「語根」× 631件

影響範囲: ページ全体が「壁紙化」する。個々の重量は中程度でも、繰り返しで閾値を超える。

### Type 4: 水平ノイズ（Horizontal Noise）

同一行に複数の視覚的重量のある要素が並ぶ。  
例: Concept Insight のチップ4個 + 4個が2行にわたって横に並ぶ

影響範囲: 視線が横に走り、縦への自然な流れを妨げる。

## 5. 静寂の設計原則

このプロダクトにとっての「静寂」は単なる余白ではなく、  
**「聖書を読む時間のための間」**。

| 原則 | 内容 |
|------|------|
| 聖書本文の前後に静寂を置く | 読む前の「息継ぎ」と読んだ後の「余韻」 |
| テーマ語の間の静寂を守る | Discovery Lobby のリズムを検索結果にも |
| ボタンは静寂の後に | テキストを読み終えた後にアクションが現れる |
| ノイズは「読む」の邪魔をしない場所に | 左サイドバーに収める（そして左を軽くする）|
