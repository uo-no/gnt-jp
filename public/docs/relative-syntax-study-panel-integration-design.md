# 関係詞 Syntax → StudyPanel Integration 責務設計(Stage K-5)

策定日: 2026-07-20
位置づけ: 関係詞 Syntax の構造情報(relativeSyntax: strong/role/referent)を、**ギリシャ語語順を
変更せず** StudyPanel で読解可能にするための責務設計。
**プロダクト方針の確定**: 「翻訳するための Syntax」ではなく **「読むための Syntax」**。StudyPanel は
ギリシャ語語順・原文構造を保持し、日本語は**構造理解を補助**するために用いる。
根拠: K-3(getRelativeSyntax)・K-4(relativeSyntax 保持・バイト等価)・relative-syntax-phase3-design.md
(語順再構成=**旧設計**として参照のみ・本設計が方針上位)。

**本文書は責務設計のみ。コード・擬似コード・実装は含まない。**
reading-engine.js / bible_data / Morph Registry / Semantic / PhraseRenderer は変更しない。
**語順再構成・自然日本語化・翻訳エンジン化は行わない。**

> **成功条件**: 「日本語として自然にすること」**ではない**。**ギリシャ語の構造・語順を保持したまま、
> 読者が関係詞と先行詞の関係を理解しやすくなること**を成功条件とする。

---

## 0. 前提と方針の位置づけ

- 旧 Phase 3(relative-syntax-phase3-design.md)は **語順再構成(関係節を先行詞の前へ)** を前提と
  していたが、現在の方針「読むための Syntax」では **語順を変えない**。本設計がこれを上書きする。
- **語順を変えない利点**: 複雑関係節(遠隔・後置・入れ子・解決不可 65)でも、再構成の誤り・自然性
  劣化のリスクがない。**構造注釈(役割・先行詞リンク)は単純節・複雑節の双方に安全に適用できる**
  (再構成と異なり複雑節を除外する必要が小さい)。
- プロジェクト核心(固定 japanese をギリシャ語構造反映の「読む日本語」へ・翻訳しない)と一致する。

## 1. 責務境界(Syntax / StudyPanel / Presentation / PhraseRenderer)

| 層 | 担当 | しないこと |
|---|---|---|
| **Syntax** | 関係詞-先行詞の構造関係・referent(verseId)解決による構造リンク・role・節/関係構造情報を**保持・提供**。構造的事実のみ | 日本語文字列生成・語順変更 |
| **StudyPanel / Presentation** | Syntax の構造情報を**表示へ反映**。関係詞と先行詞のリンクを視覚的に示す・role を読解補助として表示・**ギリシャ語語順を保持**・必要な日本語補助表示を設計 | 語順変更・自然日本語化・意味断定 |
| **PhraseRenderer** | **語順再構成に使用しない**。自然日本語化のための関係節前置を行わない。**現行責務を拡張しない** | 関係節の前置・再構成 |
| **Semantic** | (対象外) | whoever/whatever/数量/不定/自由関係・意味推論・role からの意味推論・文脈断定 |

- **Syntax は文字列を生成せず語順も変えない**(K-3/K-4 と同じ・構造情報の供給者)。
- **StudyPanel が表示を決定**する(構造情報を読解補助として可視化)。ギリシャ語語順は保持。

---

## 2. relativeSyntax の各情報の利用責務

K-4 で resolve 結果に付帯される relativeSyntax = { strong, role, referent } の利用責務を定義する。

| 情報 | 内容 | 利用責務(StudyPanel 表示) | 禁止 |
|---|---|---|---|
| **strong** | 関係詞 lemma(G3739 ὅς / G3748 ὅστις / G3745 ὅσος) | 関係詞であること・種別の識別(表示の要否判定) | 意味推論 |
| **role** | 節内役割(s 主語 / o 目的語 / io 間接目的 / p 補語 / adv 副詞 / o2) | **節内役割の読解補助ラベル**として表示(主語/目的語 等) | 助詞生成・語順変更・role からの意味推論 |
| **referent** | 先行詞リンク(verseId → 先行詞トークン) | **先行詞の可視化**(この関係詞がどの語を指すか) | 先行詞の推論生成・並べ替え |

- role は **役割の「表示ラベル」**として使う(助詞や語順にはしない)。
- referent は **先行詞への「リンク表示」**として使う(先行詞を生成・移動しない・Syntax が解決済みの
  リンクを渡す)。

---

## 3. 「ギリシャ語語順を見せるための日本語補助」の考え方

- **Reading Japanese(逐次の読む日本語)はギリシャ語トークン順のまま**(者/もの 等・語順不変)。
- **構造の補助情報(role・先行詞)は別枠で提示**する。読む日本語の本文や語順は変えない。
- 目的: 読者がギリシャ語語順で読み進めながら、**「この関係詞(〜する者)は先行詞『福音』を指し、
  節の目的語である」** という構造を補助情報から把握できるようにする。
- **日本語補助は「翻訳」ではなく「構造の道しるべ」**。自然な訳文へ再構成しない。ギリシャ語構造を
  保ったまま関係の理解を助ける。

### 補助の性格

| 補助 | 内容 | 例 |
|---|---|---|
| 節内役割 | role の読解ラベル | 「節内役割: 目的語」 |
| 先行詞リンク | referent の指す語 | 「先行詞: 福音(εὐαγγέλιον)」 |
| 構造関係 | 関係詞→先行詞の指示 | 「〜する者 → 福音 を指す」 |

- これらは **StudyPanel 内の補助表示**であり、**Reading Flow の語順・チップ本文は変えない**。

---

## 4. StudyPanel で関係詞構造を表示する最小単位

関係詞トークンについて表示する **最小構造単位**を次で定義する。

| 最小単位 | 内容 | 由来 |
|---|---|---|
| **節内役割ラベル** | 主語 / 目的語 / 間接目的 / 補語(role の読解語) | relativeSyntax.role |
| **先行詞参照** | 先行詞の語(表層)+ その読む日本語(getDisplayLabel) | relativeSyntax.referent(verseId 解決) |

- **最小単位 = {節内役割ラベル, 先行詞参照}**。これ 1 組で「この関係詞は何を指し、節でどの役割か」が
  読解できる。視覚的リンク(関係詞↔先行詞のハイライト等)は表示強化の任意要素。
- role が未注釈('-')または referent 未解決(自由関係・解決不可)の場合は、**該当項目を出さない**
  (安全 fallback・自由関係は Semantic 領域で対象外)。

---

## 5. 現行 UI の chip⇔panel 関係を壊さない前提

- **Reading Flow の chip(ギリシャ語順・者/もの 本文)は不変**。relativeSyntax は K-4 でバイト等価に
  付帯済で、**chip gloss・panel reading の文字列を変えない**。
- StudyPanel への構造表示は **既存の読む日本語・gloss に対する「追加情報」**であり、置換ではない。
- したがって **chip⇔panel 一致(reading の一致)100% は維持**される(追加表示は reading 比較の対象外)。
- **前提の明記**: 本統合は「reading(chip/panel の読む日本語)を変えず、StudyPanel に構造補助を足す」。
  reading を変える設計は採らない。

## 6. 実装開始 Gate

| Gate | 判定条件 |
|---|---|
| **G1** | K-4 の relativeSyntax(strong/role/referent)が resolve 結果で取得可能(充足) |
| **G2** | referent→先行詞の **verseId 索引解決**が用意でき、先行詞の語・読む日本語を取得できる(実測 93.9% 解決) |
| **G3** | StudyPanel に構造補助を表示する枠(reading を変えない追加表示スロット)が定義できる |
| **G4** | **chip⇔panel の reading 一致 100% を壊さない**(構造表示は追加情報・reading 不変) |
| **G5** | **語順変更・自然日本語化・Syntax の文字列生成・PhraseRenderer 拡張をしない**ことが担保される |
| **G6** | QA(構造表示の正当性・reading 不変・自由関係/未解決の非表示)の手順が用意されている |

- G1〜G6 充足で StudyPanel Integration 実装へ進む。

## 7. QA 方針

| 項目 | 方針 |
|---|---|
| relativeSyntax 取得率 | 束縛関係 1,071 で strong/role/referent が取得できること(K-4 と同一) |
| 先行詞解決率 | verseId 索引で先行詞トークン・読む日本語が取得できること(実測 93.9%) |
| **reading 不変(バイト等価)** | Reading Japanese・chip gloss・語順が K-4 と完全一致(構造表示は reading を変えない) |
| chip⇔panel 一致 | reading の一致 100%・破損形 0 を維持 |
| 構造表示の正当性 | 節内役割ラベル・先行詞参照が relativeSyntax・注釈と一致 |
| 自由関係/未解決の非表示 | referent 無 587・未解決 65・role 無を構造表示しない(安全) |
| **成功条件(構造理解)** | 自然な日本語かではなく、**ギリシャ語語順を保ったまま関係詞↔先行詞の関係が把握できるか**で評価 |

- **「日本語として自然か」は測定しない**。測定するのは「構造・語順を保持したまま関係の理解を
  助けるか」。

## 8. FROZEN 条件

本設計を FROZEN とする条件:

- Syntax / StudyPanel / Presentation / PhraseRenderer の責務境界が確定している(§1)。
- relativeSyntax(strong/role/referent)の利用責務が確定している(§2)。
- 「ギリシャ語語順を見せるための日本語補助」の考え方が確定している(§3)。
- StudyPanel で関係詞構造を表示する最小単位が確定している(§4)。
- chip⇔panel を壊さない前提が明記されている(§5)。
- Gate(§6)・QA 方針(§7)・扱わない範囲(§9)が定義されている。
- **語順再構成・自然日本語化・翻訳エンジン化・Syntax の文字列生成を含まない**。

---

## 9. Phase 3 / 本統合で扱わない範囲(明示)

| 項目 | 理由 |
|---|---|
| **語順再構成 / 関係節の前置 / 自然日本語化** | 方針(読むための Syntax)。ギリシャ語語順を保持する |
| **翻訳エンジン化** | 方針外。翻訳モードの日本語語順改善と StudyPanel 構造表示を混同しない |
| **PhraseRenderer による自然日本語化** | 現行責務を拡張しない |
| whoever / whatever / 数量(ὅσος)/ 不定(ὅστις)/ 自由関係(referent 無 587) | **Semantic 領域** |
| 関係詞の意味推論・role からの意味推論・文脈による意味断定 | **禁止**(構造の読み取り・表示のみ) |
| **referent からの推論による先行詞生成** | 禁止(Syntax が verseId 解決済みリンクを渡す) |
| **Syntax による日本語文字列生成 / 語順変更** | 禁止 |
| **Morph Rule Engine v1・Morph Registry・bible_data の変更** | 禁止(頭語 者/もの・7 lemma・データは不変) |
