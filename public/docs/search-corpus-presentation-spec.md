# Phase UX-9.9 — Corpus Presentation Specification

**作成日**: 2026-07-14  
**種別**: 仕様書（コード変更なし）  
**参照**: `docs/search-corpus-boundary-audit.md` / `docs/search-design-system.md`  
**方針**: 何を実装するかを定義する。どう実装するかは実装フェーズに委ねる。

---

## 概要

聖書原語リーダーの検索 corpus は NT（新約聖書・137,741 tokens）と LXX（七十人訳聖書・479,623 tokens）の混在構成である。UX-9.8 監査により、κύριος（91% LXX）・διαθήκη（90% LXX）等の神学的核心語で「想定した検索対象と大きく異なる結果」が提示される BLOCK 問題が確認された。

この仕様書は、その問題を解決する**表示仕様**を定める。コード変更は定義しない。

## §1. Corpus Hierarchy（コーパス階層定義）

### 1-1. 一次 corpus と二次 corpus

| 属性 | 一次 corpus | 二次 corpus |
|------|------------|------------|
| 識別 ID | `NT` | `LXX` |
| 呼称 | 新約聖書 | 旧約ギリシア語訳 |
| 略称 | NT | LXX |
| 書物数 | 27 | 39 |
| トークン数 | 137,741 | 479,623 |
| 占有率 | 22.3% | 77.7% |
| 書物キー | MAT・MRK・LUK・JHN・ACT・ROM・1CO・2CO・GAL・EPH・PHP・COL・1TH・2TH・1TI・2TI・TIT・PHM・HEB・JAS・1PE・2PE・1JN・2JN・3JN・JUD・REV | GEN・EXO・LEV・NUM・DEU・JOS・JDG・RUT・1SA・2SA・1KI・2KI・1CH・2CH・EZR・NEH・EST・JOB・PSA・PRO・ECC・SNG・ISA・JER・LAM・EZK・DAN・HOS・JOL・AMO・OBA・JON・MIC・NAH・HAB・ZEP・HAG・ZEC・MAL |

### 1-2. 位置づけ

| corpus | 役割 | デフォルト表示 |
|--------|------|-------------|
| NT | 検索の主対象。このアプリが扱う聖書テキストの本体 | **デフォルト表示** |
| LXX | 比較参照 corpus。NT 著者が参照した聖書テキスト。NT 語彙の旧約的背景を補足する | **ユーザーの操作で追加表示** |

### 1-3. 根拠

1. アプリは「新約聖書のギリシャ語を読む」ために設計されている（§1 Mission）
2. 既存アーキテクチャが `IDX.ready`（NT 構築完了）→ `IDX.l3Ready`（NT+LXX 完全完了）の二段階設計を採用しており、「NT が主・LXX が補足」という設計意図が既に実装されている
3. LXX は資産価値の高いデータであり、破棄しない（案 A を採用しない理由）

### 1-4. 呼称の原則

| 状況 | 表示形式 |
|------|---------|
| ユーザー向け簡略形 | 「新約聖書」「旧約ギリシア語訳」 |
| badge・chip など狭いスペース | 「NT」「LXX」 |
| ヘルプ・説明文 | 「旧約ギリシア語訳（七十人訳聖書 / LXX）」 |

## §2. Search Result Ordering（表示順序規則）

### 2-1. 基本規則

MUST: NT 結果を LXX 結果より先に表示する。  

現行の canonical ソート順（MAT → REV → GEN → MAL）がこの規則を満たしている。変更不要。

### 2-2. コーパス境界の明示

MUST: 結果リスト内で NT と LXX の境界を視覚的に明示する。

現行は NT 結果と LXX 結果が連続して表示され、ユーザーは境界に気づかない。  
以下の 2 案のいずれかを採用する。

**案 C-1: 折りたたみ（推奨）**

```
[ NT 714件 の結果 ]        ← NT 結果のみ表示（デフォルト）
  マタイ 1章20節 ...
  マタイ 3章3節  ...
  ...
  
[ ▶ 旧約ギリシア語訳（LXX）の結果を表示  7,458件 ]   ← クリックで展開
```

- デフォルト: LXX 結果は折りたたみ（非表示）
- ユーザーが展開ボタンをクリックすると LXX 結果が後方に追加される
- 展開ボタンのラベルには LXX 件数を表示する

**案 C-2: 後方セクション区切り**

```
[ NT 714件 の結果 ]
  マタイ 1章20節 ...
  ...（全714件）

  ─── 旧約ギリシア語訳（LXX）7,458件 ─────────

  創世記 1章1節 ...
  ...
```

- NT 結果と LXX 結果を区切り線で分離
- LXX 結果は常時表示（折りたたみなし）
- 区切り行にセクション名と件数を表示

### 2-3. 採用方針

**案 C-1 を推奨する。** 理由:

- LXX が全体の 77.7% を占めるため、常時表示（案 C-2）では LXX が主役になる
- 大多数の初心者ユーザーは NT のみを期待している
- 既存の NT-first ローディングアーキテクチャと整合する
- 研究者は折りたたみを展開すれば LXX にアクセス可能

## §3. Result Count Display（件数表示仕様）

### 3-1. 基本フォーマット

現行: `{total} 件`（単一値）

新仕様:

| 状態 | 表示形式 | 例 |
|------|---------|---|
| NT のみ（LXX=0） | `{nt} 件` | `714 件` |
| NT+LXX 両方あり | `NT {nt}件 + LXX {lxx}件` | `NT 714件 + LXX 7,458件` |
| NT=0, LXX>0 | `LXX {lxx}件（新約聖書は 0 件）` | `LXX 7,458件（新約聖書は 0 件）` |

### 3-2. 表示位置

現行の `result-count` 要素（`id="result-count"`）に表示する。  
補足情報が増えるため、2行レイアウトが必要な場合は `result-count` 内で `<span>` を分割する。

### 3-3. 件数の計算タイミング

- NT 件数: `IDX.ready` 完了後（NT のみ）
- LXX 件数: `IDX.l3Ready` 完了後（差分）
- 表示: `IDX.ready` 完了時点で NT 件数を先行表示。l3Ready 完了後に LXX 件数を追加

## §4. Hit Card Corpus Labeling（ヒットカードのコーパス識別仕様）

### 4-1. 最低限要件（P1）

MUST: 各 `.hit-card` 要素に `data-corpus` 属性を付与する。

```html
<!-- NT カード -->
<div class="hit-card" data-corpus="NT">

<!-- LXX カード -->
<div class="hit-card" data-corpus="LXX">
```

目的:
- CSS による corpus 別スタイリングを可能にする
- JavaScript によるフィルタリングを可能にする
- テストスクリプトでの corpus 判定を可能にする

### 4-2. 推奨視覚要件（P2）

SHOULD: LXX カードに corpus badge を表示する。NT カードは無標（badge なし）。

```
┌─────────────────────────────────────────┐
│ 詩篇 2章2節   [LXX]                     │   ← LXX badge
│ ─────────────────────────────────────── │
│  κύριον τὸν θεὸν αὐτῶν               │
│ ─────────────────────────────────────── │
│  (翻訳なし)                               │   ← japanese=null 時
│ ─────────────────────────────────────── │
│                          [ 本文で読む ] │
└─────────────────────────────────────────┘
```

**badge デザイン仕様:**

| 属性 | 値 |
|------|---|
| テキスト | `LXX` |
| 位置 | `hit-ref` の右隣（inline） |
| サイズ | `--text-caption`（0.75rem） |
| 色 | `--text-sub`（通常テキストより控えめ） |
| 背景 | `--bg-subtle` または `transparent` |
| border | `1px solid var(--border)` |
| padding | `1px 4px` |
| border-radius | `var(--radius-s)`（6px） |
| font-weight | 通常（400） |

**NT カード**: badge なし。「NT」表示は不要（デフォルト corpus は無標とする）。

### 4-3. 日本語グロス欠損への対応

LXX トークンは `japanese: null`（グロスデータなし）。

| 状況 | 現行 | 新仕様 |
|------|------|--------|
| LXX カードの翻訳テキスト欄 | 空欄（見た目が壊れる） | `（翻訳テキストなし）` と表示 |
| 表示スタイル | — | `--text-sub` 色, italic |

### 4-4. 変更しない要素

| 要素 | 変更しない理由 |
|------|-------------|
| `hit-ref` の日本語書名（「詩篇2章2節」等） | 既存の BOOK_SHORT_SEARCH マップが LXX 書名を正しく保持している |
| `ht-badge`（語根・近接・品詞バッジ） | hit type バッジは corpus と無関係 |
| `本文で読む` ボタン | LXX の箇所も `index.html` でナビゲート可能 |
| snippet テキスト（ギリシャ語コンテキスト） | corpus によらず表示 |

## §5. Distribution Panel Corpus Separation（分布パネルのコーパス分離仕様）

### 5-1. 現状の問題

現行 `_renderBookDist()` は全 corpus の書物を頻度降順で最大 12 件表示する。  
κύριος 検索時、上位 12 書物がすべて LXX 書物になり、NT 分布が不可視になる。

### 5-2. 新構造

分布パネルを NT セクションと LXX セクションに分割する。

```
出現分布（書物別）
┌─ 新約聖書 ────────────────────────────────────
  使徒行伝   107 ████████████
  ルカ       102 ████████████
  マタイ      80 █████████
  ヨハネ      75 ████████
  ローマ      45 █████
  ヘブル      32 ████
  [他 9 書物]

┌─ 旧約ギリシア語訳（LXX） ─────────────────────
  詩篇       818 █████████████████████████
  エレミヤ   607 ██████████████████
  申命記     558 █████████████████
  エゼキエル  487 ███████████████
  歴代誌Ⅱ   429 █████████████
  イザヤ     424 █████████████
  [他 33 書物]
```

### 5-3. セクション仕様

| 属性 | NT セクション | LXX セクション |
|------|-------------|--------------|
| 見出し | `新約聖書` | `旧約ギリシア語訳（LXX）` |
| ソート | 頻度降順 | 頻度降順 |
| 最大表示 | 8 書物 | 8 書物 |
| 超過時 | `他 N 書物` | `他 N 書物` |
| ゼロ件時 | セクション非表示 または `0 件` を表示して折りたたみ | 同左 |
| クリック動作 | 書物絞り込み（現行と同じ） | 同左 |

### 5-4. バーグラフの相対スケール

| 選択肢 | 説明 | 推奨 |
|--------|------|------|
| A: セクション内相対 | 各セクションの最大値を 100% とする | ✅ 推奨 |
| B: 全体相対 | 全 corpus の最大値を 100% とする | NT バーが細すぎる問題がある |

**採用: A（セクション内相対スケール）**  
理由: LXX 詩篇 818 件を基準にすると、NT 使徒 107 件のバーが 13% の細さになり判読不能になる。

### 5-5. セクションヘッダーの視覚仕様

| 属性 | 仕様 |
|------|------|
| クラス | `dist-section-title`（新規） |
| font-size | `--text-caption`（0.75rem） |
| font-weight | 700 |
| letter-spacing | 0.10em |
| text-transform | uppercase |
| color | `--text-sub` |
| opacity | 0.6 |
| margin-top | `--space-sm`（8px）（最初の NT セクションは 0） |
| margin-bottom | `--space-xs`（4px） |

`dist-section-title` は既存の `.dist-title`・`.ci-section-title` と同じ Section Label パターン（Design System §4-6）に準拠する。新規トークン追加なし。

## §6. Corpus Filter UI（コーパスフィルター仕様）

### 6-1. デフォルト状態

**デフォルト: NT のみ検索**

```
┌──────────────────────────────────────────────────┐
│  検索語   [ ________ἀγαπάω_________ ] [🔍]       │
│  ▶ 詳細検索                                       │
└──────────────────────────────────────────────────┘
```

- 検索実行時: NT corpus のみ対象（`IDX.ready` ベース）
- 件数表示: `353 件`（NT のみ）

### 6-2. LXX 展開状態

詳細検索（Advanced Layers）内に「LXX を含む」トグルを追加する。

```
┌──────────────────────────────────────────────────┐
│  検索語   [ ________κύριος____________ ] [🔍]    │
│  ▼ 詳細検索                                       │
│  ┌────────────────────────────────────────────┐  │
│  │ 語形   フレーズ   近接   品詞               │  │
│  │ ─────────────────────────────────────────  │  │
│  │ ☑  旧約ギリシア語訳（LXX）を含む          │  │  ← 新規
│  └────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

- トグル OFF: NT のみ、`714 件`
- トグル ON: NT+LXX、`NT 714件 + LXX 7,458件`

### 6-3. トグルの仕様

| 属性 | 仕様 |
|------|------|
| 要素 | `<input type="checkbox">` または `<details>` 内 toggle |
| デフォルト | OFF（チェックなし） |
| ラベル | `旧約ギリシア語訳（LXX）を含む` |
| 位置 | Advanced Layers パネル内、語形/品詞フィルターの後 |
| 永続化 | `localStorage` で保存（オプション; 実装フェーズで判断） |

### 6-4. LXX トグル ON 時の完全な表示仕様

LXX トグル ON 時、以下の全 UI が §3〜§5 の仕様に従い corpus 情報を表示する:

- 件数: `NT N件 + LXX N件` 形式（§3）
- Hit Cards: LXX カードに `data-corpus="LXX"` + LXX badge（§4）
- Hit Cards: NT 結果 → LXX 折りたたみセクション（§2）
- 分布パネル: NT セクション + LXX セクションの 2 分割（§5）

LXX トグル OFF 時（デフォルト）は既存 UI と同一（corpus 情報の表示なし）。

## §7. Empty / Zero Hit Behavior（ゼロ件・欠損状態の仕様）

### 7-1. 状態マトリクス

| 状態 | NT 件数 | LXX 状態 | 表示 |
|------|---------|----------|------|
| A. 完全ゼロ | 0 | OFF or 0 | 現行の zero-hits UI（変更なし） |
| B. NT ゼロ, LXX あり（トグル OFF） | 0 | OFF（未検索） | 現行の zero-hits UI + LXX ヒント |
| C. NT ゼロ, LXX あり（トグル ON） | 0 | 7,458 | LXX 結果を表示 + corpus 明示 |
| D. NT あり, LXX ゼロ（トグル ON） | 714 | 0 | NT 結果のみ。LXX セクション = 0 件で折りたたみ |
| E. NT あり, LXX あり | 714 | 7,458 | §2〜§5 の仕様に従い表示 |

### 7-2. 状態 B（NT=0, LXX OFF）の仕様

```
∅
「{query}」に該当する新約聖書の箇所がありませんでした
                            ↑ 「箇所」の前に「新約聖書の」を追加

[ヒント] 旧約ギリシア語訳（LXX）には N 件の用例があります。
         詳細検索から「LXX を含む」を選択すると表示できます。
```

条件: このヒントは `IDX.l3Ready === true` かつ LXX 件数 > 0 の場合のみ表示する。

### 7-3. 状態 D（NT あり, LXX セクション = 0 件）の仕様

- 分布パネルの LXX セクション: セクションヘッダーのみ表示、`0 件` または非表示
- 件数表示: `714 件`（LXX=0 の場合、LXX を件数に含めない）

### 7-4. Zero Hit 完全ゼロ（状態 A）の変更なし項目

| 要素 | 変更 |
|------|------|
| `∅` アイコン | なし |
| メッセージ文言 | 状態 B 以外はなし |
| Recovery チップ（愛・祈り・救い・キリスト） | なし |
| `zero-hint` テキスト | なし |

## §8. Concept Insight との関係

### 8-1. Concept Insight は corpus に依存しない

`SEARCH_CONCEPTS` データは NT 神学概念を基盤として設計されている。  
Concept Insight が発火する条件（日本語概念マッチ）は corpus 設定と無関係である。

### 8-2. LXX 結果との意味的ギャップ

κύριος を検索した場合、Concept Insight は「主（キリスト論的文脈）」の解説を表示するが、実際の検索結果の 91% は詩篇・申命記等 LXX 書物の「ヤハウェ（主）」用例である。

この意味的ギャップは認識されているが、Concept Insight のロジック変更は禁止対象のため、本仕様では対処しない。

将来的な対処として:  
LXX トグル OFF（NT のみ）の場合は Concept Insight と結果の整合性が保たれるため、デフォルト NT 表示がこの問題を最小化する。

### §11. Corpus Visibility Rules — コーパス可視性原則

#### 11-1. 件数の内訳明示

複数の corpus が検索対象に含まれる場合、結果ヘッダーの件数表示に corpus 別内訳を明示しなければならない。

> 例: `NT 714件 + LXX 7,458件`（`8,172 件` だけでは不可）

#### 11-2. 分布パネルのコーパス境界

分布パネル（Distribution Panel）は、corpus をまたぐ場合にセクション境界を設けなければならない。NT 書物と LXX 書物を同一の頻度リストに混在させてはならない。

#### 11-3. Hit Card のコーパス識別子

すべての Hit Card に `data-corpus` 属性を付与しなければならない。  
`data-corpus="NT"` または `data-corpus="LXX"` のみを有効値とする。  
LXX カードは視覚的な corpus badge を表示することを推奨する（SHOULD）。  
NT カードは無標とする（badge 不要）。

#### 11-4. デフォルト corpus の宣言

このアプリの一次 corpus は **NT（新約聖書）** である。  
検索のデフォルト対象は NT とする。LXX はユーザーの操作によって追加する。

#### 11-5. LXX の位置づけ

LXX は「比較参照 corpus」として位置づける。  
NT テキストの語彙的・神学的背景を補足する情報源であり、「新約聖書検索の代替」ではない。  
LXX 結果は NT 結果の後方に、明示的な境界とともに提示する。

#### 11-6. 日本語グロス欠損の扱い

LXX トークンに日本語グロスが存在しない場合（`japanese: null`）、空欄を表示してはならない。  
「（翻訳テキストなし）」等の明示的なプレースホルダーを表示しなければならない。

#### 11-7. 追加禁止

Corpus を識別するための新規カラートークン・新規フォントサイズ・新規 badge クラスを追加する場合、既存デザイントークンで代替できないことを証明しなければならない。  
`--text-caption`・`--text-sub`・`--radius-s` の組み合わせで badge は実現可能である。

## §10. 実装ロードマップ（優先度）

本仕様の実装を以下の順序で実施することを推奨する。

| フェーズ | 内容 | 仕様参照 | 規模 | 優先度 |
|---------|------|---------|------|--------|
| UX-9.9-A | Hit Card に `data-corpus` 属性付与 | §4-1 | S | P1 必須 |
| UX-9.9-B | 件数表示の corpus 内訳追加 | §3 | S | P1 必須 |
| UX-9.9-C | 分布パネルの NT/LXX 2セクション分割 | §5 | M | P1 必須 |
| UX-9.9-D | LXX Hit Card に badge 表示 | §4-2 | M | P2 推奨 |
| UX-9.9-E | LXX 結果の折りたたみセクション | §2 案 C-1 | L | P2 推奨 |
| UX-9.9-F | Advanced Layers に LXX トグル追加 | §6 | L | P2 推奨 |
| UX-9.9-G | Zero hit corpus ヒント表示 | §7-2 | S | P3 推奨 |
| UX-9.9-H | Design System §11 追加 | §9 | XS | P3 推奨 |

### 最小実装（MVP）

UX-9.9-A + UX-9.9-B + UX-9.9-C の 3 点が MVP である。  
この 3 点のみで BLOCK 問題（κύριος・διαθήκη の分布パネル乗っ取り問題）を解決できる。

### 実装不可の禁止事項

本仕様のいかなる実装においても以下を変更してはならない:

- `resolveTerm` / `resolveConcept`
- Pattern Engine / Wallace Engine
- `search-concepts.json`
- `index.html` / StudyPanel
- Knowledge Graph の内部構造

## §11. 仕様決定の根拠サマリー

| 決定 | 根拠 |
|------|------|
| Case C（NT デフォルト + LXX トグル）採用 | LXX は 77.7% を占め、トグルなしでは LXX が主役になる。案 A（NT のみ）は LXX データ資産を捨てる |
| NT カードを無標（badge なし）とする | デフォルト corpus は無標が原則（P-09: 専門語は奥へ）。NT badge はノイズになる |
| セクション内相対スケール採用 | LXX 詩篇818を基準にすると NT 使徒107が 13% の細さになり判読不能 |
| LXX badge を「LXX」のみとする（日本語長称なし） | badge の役割は識別であり説明ではない。「七十人訳聖書」は hint/tooltip に委ねる |
| LXX グロス欠損は空欄不可 | 空欄は「データあり、翻訳なし」と「データなし」を区別できない。明示的プレースホルダーが必要 |
