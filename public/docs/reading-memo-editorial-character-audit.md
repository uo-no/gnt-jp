# Reading Memo Editorial Character Audit(Stage RM-0)

実施日: 2026-07-22
位置づけ: 現「この節の読書メモ」(Wallace Greek Grammar Beyond the Basics 由来の自動生成)に、将来
Reading Japanese の観察情報(同一 lemma 反復・語族・別 lemma 同訳・修辞反復 等)を追加する案がある。
**実装ではなく「現在の読書メモの役割・文体・編集思想を壊さないか」を監査する。**
**コード変更・UI 変更・実装提案は一切禁止。監査のみ。**
読む対象: reading memo 生成処理(reading-projection.js)・生成テンプレート(clause-analyzer.js
ReadingFormatter / _WALLACE_TEXT)・Wallace 由来情報・実際の出力。

Severity 凡例: **P0(致命)/ P1(高)/ P2(中)/ P3(低)**。

---

## 監査で確認した現状(基準)

- **自然文生成源は ReadingFormatter ただ一つ**(clause-analyzer.js §6・Phase 12 でアーキテクチャ固定)。
  reading-projection.js は**純粋データを渡すだけ**(文章生成・説明・表示判断をしない・v1.0.0)。
- テンプレートは `_WALLACE_TEXT`(discourse 関係 → 「静かな 1 文」)。例:
  - DISCOURSE_EXPLANATION: 「ここでは、前の内容がなぜ成立するのかが説明されています。」
  - CONTENT: 「ここでは、その内容が具体的に明らかにされています。」
  - CONDITION: 「ここでは、ある条件が前提として提示されています。」
- **厳格な文体規律**(clause-analyzer.js 1011–1099):
  - 主語は常に**読書の視点**(「ここでは/この流れでは/この箇所では」)。**「この節は」等の分析主語は禁止**。
  - **分類ラベル(discourse.type)・生マーカー(γάρ/ὅτι)・confidence 数値を一切露出しない**。
  - **Guard Rule `assertReadingTextSafe()`**: 内部トークンや数値(`\d+%`)が混入すると **throw**(黙って
    除去せず生成側で即失敗)。

## Phase 1: 読書メモの目的(割合)

| 目的 | 割合 | 評価 |
|---|---|---|
| **読解補助** | **~80%** | 中核。談話の流れ(なぜ/内容/目的/結果/条件/対比)を読者が追えるようにする |
| **観察** | **~20%** | discourse 関係の観察だが、分析でなく読書の声で提示 |
| 文法説明 | **~0%** | 意図的にゼロ(文法用語・ラベルを隠す) |
| 神学説明 | 0% | なし |
| 背景説明 | 0% | なし |
| 適用 | 0% | なし |

- **評価**: **読解補助(談話の流れの観察)に純化**。文法・神学・背景・適用は**意図的に排除**。
- **Evidence**: _WALLACE_TEXT の全文が「ここでは…されています」の流れ提示(1101–1126)。文法/神学語なし。
- **Severity**: **P3**(現状は目的が明快・純粋)。

## Phase 2: Wallace 情報の特徴(扱う範囲)

| Wallace 次元 | 扱う | メモでの露出 |
|---|---|---|
| **談話(discourse)** | ✓ 中核 | 関係を読書の声で(説明/内容/目的/結果/条件/対比) |
| **構文(syntax)** | ✓ 基盤 | 節種(条件節/対比節)→ 文へ橋渡し(1055–1062) |
| 文法(grammar) | △ 基盤にあるが**非露出** | 用語は出さない(StudyPanel morph 側の責務) |
| 意味(semantics) | ✗ | メモは語義を扱わない |
| 修辞(rhetoric) | ✗ | メモは修辞構造を扱わない |

- **評価**: **Wallace の談話・構文次元のみを、文法用語を伏せて読書の声に翻訳**。意味・修辞は扱っていない。
- **Evidence**: classifyDiscourseRelation の型(γάρ/ὅτι/ἵνα/εἰ/ἀλλά…)を _WALLACE_TEXT へ写像(1044–1062)。
- **Severity**: **P3**。

## Phase 3: 文体監査

| 文体 | 該当 |
|---|---|
| **観察型** | **最も近い**(discourse 関係の静かな観察) |
| 説明型 | 部分的(「説明されています」等)だが**分析主語を避け**、教師的でない |
| 教師型 | **非該当**(「〜すべき/理解しよう」なし・命令や指導口調なし) |
| 辞書型 | **非該当**(定義・語釈を出さない) |

- **評価**: **観察型**(静かな読書の声)。**教師型・辞書型を明確に排除**した「分類ラベルを出さない静かな 1 文」。
- **Evidence**: 主語=読書の視点・分析主語禁止(1013–1014)・ラベル/数値非露出(1018–1024)。
- **Severity**: **P3**。

## Phase 4: Information Density

- **評価**: **現状は極めて低密度(意図的)**——discourse 関係につき**「静かな 1 文」のみ**。数値なし・
  ラベルなし・語釈なし。**この抑制(restraint)こそメモの identity**。
  - **Lexical Observation を追加した場合、過密の危険は高い**。反復・語族・別 lemma・出現回数を**素データや
    複数文で足すと「静かな 1 文」の性格が崩れ**、observation dump 化する。特に**出現回数・Strong・LN は
    数値/ラベルであり Guard Rule と衝突**(throw)。
- **Evidence**: _WALLACE_TEXT は 1 関係 1 文(1101–1126)。Guard Rule が数値/内部語を throw(1087–1098)。
- **Severity**: **P1**(無条件追加は密度崩壊・Guard 衝突のリスク)。

## Phase 5: Editorial Character(一文定義)

> **「分類ラベル・数値・原語マーカーを一切出さず、ギリシャ語の談話の流れを読者の視点から静かな 1 文で
> 追う、読解補助の編集メモ」。**

- **Evidence**: 主語=読書視点・非露出規律・Guard Rule・1 関係 1 文(clause-analyzer.js §6)。
- **Severity**: **P3**(人格は明確)。

## Phase 6: Boundary(各追加が編集思想を壊すか)

| 追加候補 | 性質 | 編集思想への影響 | 判定 |
|---|---|---|---|
| **同一 lemma 反復** | 流れの観察(読みで聞こえる反復) | **静かな 1 文で観察として言える** | **条件付きで壊さない** |
| **修辞反復** | 流れ/響きの観察 | 同上(「同じ響きが繰り返されています」等の観察) | **条件付きで壊さない** |
| 語族 | 語源/Token 情報 | 談話の流れでなく語詳細=**別性格**(StudyPanel L2 責務) | **壊す** |
| 別 lemma 同訳 | Corpus 逆引き | 流れでなく語彙照合=別性格(Corpus 責務) | **壊す** |
| 出現回数 | **数値** | **Guard Rule と衝突(数値 throw)**・静かな声に数を持ち込む | **壊す** |
| Strong 番号 | **識別子/ラベル** | メモはラベルを隠す=根幹規律違反 | **壊す** |
| Louw-Nida | **分類ラベル** | 分類露出禁止と衝突 | **壊す** |
| Cluster | Corpus/意味クラスタ | 流れでなくデータ=別性格 | **壊す** |
| Concordance | Corpus リスト | 静かな 1 文でなく一覧=別性格 | **壊す** |

- **評価**: **区別線は「静かな読書の声で流れとして言えるか」**。反復(lexical/修辞)は**流れの観察として
  言えるため条件付きで許容**。語族・別 lemma・出現回数・Strong・LN・Cluster・Concordance は
  **数値/ラベル/語詳細/Corpus データ**であり、**メモの性格を壊す(SP-SD-1 の Token/Corpus 責務=StudyPanel 側)**。
- **Evidence**: Guard Rule(数値/ラベル throw)・読書視点規律・SP-SD-1(word-anchored data は StudyPanel)。
- **Severity**: **P1**(ラベル/数値系は無条件追加で根幹規律違反)。

## Phase 7: 推奨事項(壊さない条件のみ・実装案なし)

**追加すべきか否かは述べない。「追加しても編集思想を壊さない条件」のみ:**

1. **同じ静かな読書の声**で書く(主語=「ここでは/この流れでは」・「この節は」等の分析主語を使わない)。
2. **数値を出さない**(出現回数・confidence 等・Guard Rule に従う)。
3. **ラベル/識別子/原語マーカーを出さない**(Strong・Louw-Nida・型名・γάρ 等)。
4. **「流れ/読みで聞こえる現象」として言える**もののみ(反復・響き)。**データ照合(Concordance/Cluster/語族)は
   メモに持ち込まない**。
5. **「静かな 1 文」の密度を保つ**(observation dump 化させない)。
6. **assertReadingTextSafe を通る**(内部トークン・数値の非混入)。
7. **語詳細・Corpus データは StudyPanel の Token/Corpus セクションへ**(SP-SD-1)——メモの談話の流れの
   identity を侵さない。

- **Severity**: **P2**(条件を満たせば壊さない・満たさなければ壊す)。

## Conclusion

- **現在の読書メモは、目的(読解補助)・文体(観察型)・編集思想(分類ラベル/数値を出さない静かな読書の声)
  が一貫し、極めて明確な人格を持つ**。Wallace の談話・構文次元のみを、文法用語を伏せて 1 文に翻訳する
  抑制の効いた設計であり、**その抑制自体が identity**。
- **Reading Japanese の観察情報の追加は、性質で二分される**:
  - **反復(同一 lemma・修辞)は「流れの観察」として同じ静かな声で言えるため、条件を満たせば壊さない**。
  - **語族・別 lemma 同訳・出現回数・Strong・Louw-Nida・Cluster・Concordance は数値/ラベル/語詳細/Corpus
    データであり、メモに載せると編集思想(非露出規律・Guard Rule・談話の流れ)を壊す**。これらは
    **SP-SD-1 の通り StudyPanel の Token/Corpus セクションの責務**。
- **したがって、追加を検討する場合の唯一の安全条件は Phase 7 の 7 条件**(静かな声・数値/ラベル非露出・
  流れとして言える現象のみ・低密度・Guard 通過・語詳細は StudyPanel へ)。**実装は本監査の対象外**。

> 追う、読解補助の編集メモ」。**
