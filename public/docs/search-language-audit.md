## 2. 問題点

### 問題 A — Rule 1 違反: 「見る」と「読む」の混在

**本文への遷移ボタンが「見る」を使っている**

- 行 3634: `本文で見る`
- §6 Language Rules: 「読む」= 本文テキストへの接続。「見る」= 視覚確認・展開（インタラクションなし）
- 本文（index.html）への遷移は「読む」行為であり、「見る」では動詞が不正確

---

### 問題 B — Rule 2 違反: 「原文」の誤解

**「原文を見る」が表示する内容は「ギリシャ語テキスト」ではない**

- 行 3649, 3683, 3684: `原文を見る` / `原文を閉じる`
- 実際に展開されるのは `.wlv-flow-stream`（日本語グロスをギリシャ語順に並べたチップ列）
- 「原文」= ギリシャ語本文テキストと誰もが解釈するが、実態は異なる
- §6 禁止ワード: 「原文を見る」→「原語の流れを見る」

---

### 問題 C — Rule 3 違反: 専門語「特徴共起」

**統計専門語が初心者の目に触れる位置にある**

- 行 3765: `特徴共起`（Lemma グループ内 `.lg-cooc-title`）
- 「特徴共起」は情報検索・統計の専門用語。初心者には意味が通じない
- §6 禁止ワード: 「特徴共起」→ 「よく一緒に出る語」
- 補足: line 1384 の `.cooc-title` は既に「よく一緒に出る語」と正しく表示されている。Lemma グループ内だけが不統一

---

### 問題 D — 動詞「調べる」の検索ページ使用

**§6 Language Rules: 「調べる」は StudyPanel 文脈。検索ページでは使用最小限**

- 行 1354: `聖書の大切なテーマから調べることができます`
- 「調べる」は詳細分析・深掘りを連想させる。検索ページの文脈は「探す」
- 変更候補: `聖書の大切なテーマから探すことができます`

---

### 問題 E — §6 禁止ワード: 「関連する検索」「関連する文法」

**内容と乖離したラベル（さらに詳しく内部）**

- 行 2602: `関連する検索`（中身: パターン検索プリセット）
- 行 2603: `関連する文法`（中身: Wallace 構文分類）
- §6 禁止ワード表に記載。「検索」「文法」では内容が予測できない
- 変更候補: 「パターンで探す」「構文で探す」

---

### 問題 F — Tooltip のみの affordance（参考情報）

**修正対象外（表示文言ではなく機能改善領域）**

- 行 2591: `title="Abbott-Smith 辞書を開く"`（hover tooltip のみ）
- タップ環境で到達できない。これは UX-9-8（affordance 追加）の対象であり今回は変更しない

| 文言 | 理由 |

| `語根で探す` | 「探す」動詞が正しい。§6 準拠 |
| `フレーズで探す` | 「探す」動詞が正しい。§6 準拠 |
| `テーマから探す` | 「探す」動詞が正しい。§6 準拠 |
| `近くに出てくる語` | 平易な日本語。変更不要 |
| `品詞・語形で絞る` | 詳細検索内。「語形」はやや専門的だが説明文がフォロー |
| `よく一緒に出る語` | 既に正しい（line 1384） |
| `さらに詳しく` | §6: Concept Insight の折りたたみ専用ラベル |
| `代表例` | 平易。変更不要 |
| `構文パターン検索` | 構文パネルのタイトル。専門ユーザー向けパネルの見出しとして許容 |
| `関連するギリシャ語` | Concept Insight 内。ギリシャ語は文脈があり適切 |
| `関連する概念` | §6 設計仕様で保留中。「関連するテーマ」への変更は検討中だが今回対象外 |
| `原語データなし` | エラーフォールバック。内部的な表示 |
| `共起語なし` | エラーフォールバック。専門語だが共起パネル文脈で許容 |
| `残り N 節を表示` | §6 準拠。「節」は聖書用語として適切 |
| `さらに表示（残り N 件）` | §6 準拠 |
| `品詞を選ぶとさらに絞り込めます` | Morph UI 内のヒント。文脈上「絞り込む」は適切 |
| `検索する`（ボタンテキスト） | 正しい動詞。変更不要 |
| `検索中…` | ローディング状態。変更不要 |
| `距離ベース共起`（title属性） | Tooltip。専門ユーザー向けの内部ラベル。変更対象外 |
| `形態素フィルタ`（title属性） | 同上 |
