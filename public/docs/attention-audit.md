# UX-11-1 — Attention Audit

**実施日**: 2026-07-15  
**種別**: 監査のみ（実装変更なし）  
**対象**: Discovery Lobby 状態 / 検索結果状態（Desktop 1280×800 / Mobile 390×844）

---

## 評価指標

| 指標 | 説明 | スケール |
|------|------|--------|
| First Fixation | 画面を開いて 0.5 秒以内に最初に見るもの | ★〜★★★★★ |
| Second Focus | 最初から次へ自然に目が動くか | ★〜★★★★★ |
| Exploration | 右パネル・下方への視線移動が起きるか | ★〜★★★★★ |
| Reading Flow | 視線が ↓↓↓ と流れるか（← が混じると崩壊）| ★〜★★★★★ |
| Completion | 最後に意図した要素（テーマ語）で視線が止まるか | ★〜★★★★★ |

## 1. Discovery Lobby 状態（Desktop）

### 現状の視線経路

```
LEFT PANEL                    RIGHT PANEL
──────────────────────────    ──────────────────────────
聖書原文検索         [🔗]     ← (3) ここまで到達しない
                              ← (4) 0.5秒以内には無視
 ✓ 準備完了

 検索語
 ┌──────────────────────┐
 │  愛・信じる・ἀγαπάω  │    ← (2) ようやく気づく
 └──────────────────────┘

 ┌──────────────────────┐
 │  🔍  検索する        │    ← (1) FIRST FIXATION
 └──────────────────────┘

 ┌──────────────────────┐
 │ ✓ 語根で探す         │    ← (2) 次に見る
 │   活用形・格変化を無視│
 └──────────────────────┘

 › 詳細検索                   今日は、              ← (5) やっと
                              どんなことばを読みますか？
                                                    ← (6) 到達するかも
                              忍耐

                              信仰

                              希望

                              証し
```

**First Fixation: 検索ボタン（意図と真逆）**

最も輝度コントラストが高く、最も大きく、最も彩度が高い要素が `run-btn`。  
背景の白、入力欄のグレー、テキスト類の低コントラストに対し、  
`var(--highlight)` 塗り + 白文字 + width:100% は視覚的な「爆発」。  
ユーザーは開いた瞬間に「このツールを使うには青いボタンを押せばよい」と刷り込まれる。

**理想の First Fixation**

```
アプリタイトル（ブランド認識）
     ↓
検索入力欄（ここに入力する）
     ↓
中央の問いかけ（今日は、どんなことばを読みますか？）
     ↓
テーマ語（忍耐 / 信仰 / 希望 / 証し）
```

### Reading Flow の詳細（現状）

```
(A) タイトル行     →
(B) 入力欄         ↓
(C) 検索ボタン     ← ← ← 大きさ・色で「最重要」と誤認
(D) 語根カード     ↓ ← ← width:100% で視線が左端に引き戻される
(E) 詳細検索       ↓
                   ↓  ← ← ここで止まる（左パネルが終わる）

右パネルへ到達:  0.5〜2.0秒後（遅すぎる）
```

**理想の Reading Flow**

```
(A) タイトル                  →  右パネル
(B) 入力欄（自然なフォーカス）   今日は、
(C) ─────────────────────────   どんなことばを読みますか？
    （左右が同時に読まれる）         ↓
                              テーマ語（ゆっくり読む）
(D) ボタン（必要な時だけ気づく）
```

---

### Exit Point — Discovery Lobby（Desktop）

| | 現状 | 理想 |
|-|------|------|
| 視線の終着点 | 「詳細検索」disclosure（または語根カード） | テーマ語「証し」 |
| 終着の質 | 機能的（道具に終わる） | 問いかけ的（意味に終わる） |
| 次の行動への誘引 | 「検索せよ」という圧力 | 「読んでみたい」という引力 |

Discovery Lobby の本質は「引力」。しかし現在の Exit Point は「圧力（ボタン）」で終わっている。

## 4. 理想の視線設計

### Discovery Lobby（Desktop の理想）

```
視線の流れ（理想）

[聖書原文検索]           今日は、
                    ↓    どんなことばを読みますか？
[入力欄]            →
                         ↓
[語根（控えめ）]         忍耐
[詳細検索（text）]  →
                         ↓
                         信仰

                         ↓

                         希望

                         ↓

                         証し  ← EXIT（引力で終わる）
```

条件:
- 左パネルは「存在するが主張しない」
- First Fixation が右パネルか入力欄に来るように左の重量を落とす
- Exit Point が「テーマ語」になるよう右パネルに重力を持たせる

### 検索結果（Desktop の理想）

```
愛                 631件
新約 / LXX
─────────────────────────
関連語（静かに）
─────────────────────────
マタイ5章43節

『隣り人を愛し、敵を憎め』と言われていたことは、
あなたがたの聞いているところである。
                                    › 本文で読む（ghost）
─────────────────────────
マタイ5章44節

しかし、わたしはあなたがたに言う。
                                    › 本文で読む（ghost）
```

「本文で読む」が ghost になることで、ユーザーはまずテキストを読む。  
興味を持った節だけ「本文で読む」を探して押す。  
この順序が Reading Flow の正常化。
