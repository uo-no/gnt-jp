## Current Flow

```
初回訪問
 ↓
Empty State
 ├── placeholder: 愛・信じる・ἀγαπάω …
 ├── 例: [ἀγαπάω][πιστεύω][G3056]        ← ギリシャ語専用
 ├── テーマから探す: [愛][祈り][信仰][救い][罪][キリスト][十字架][復活]
 └── 文法パターン: [互いに＋命令形][μή＋命令形][ἵνα＋接続法][ἐάν＋接続法]

入力 → 検索
 ├── 日本語: Concept完全一致 → Insight表示 + 語根検索
 ├── 日本語: 非登録語 → 日本語部分一致検索（日本語サジェスト表示）
 ├── ギリシャ語: lemma検索（Insight発火しない）
 ├── Strong番号: 検索実行 → [要調査: 全ケースで0件]
 └── 英語/その他: text型検索 → 多くの場合0件

0件時
 └── ∅ 「X」に該当する箇所がありませんでした  ← 次のアクション提示なし
```

## UX 問題分析

### A. 「テーマから探す」だけで十分か

**判定: ほぼ十分だが0件時の橋渡しがない**  
初回訪問時は8テーマが明示されており、日本語初心者にとって十分な入口となっている。ただし、検索して0件になった後にテーマチップへ戻る導線がない。

### B. 入力欄フォーカス時に次の行動が分かるか

**判定: GOOD**  
Placeholder「愛・信じる・ἀγαπάω …」が3つの入力形式を示しており、初見でも日本語入力が有効と理解できる。

### C. 0件時に次の検索方法が提示されるか

**判定: IMPROVE**  
現状は「該当する箇所がありませんでした」のみ。「テーマから探す」の再提示、類似語の提案、検索形式のヒントがない。特に「神様」「お祈り」のような一般的な日本語で0件になるケースでの救済策が必要。

### D. Concept検索と通常検索の関係が理解できるか

**判定: IMPROVE**  
Insightカード（「愛」について）が表示されても、それが「概念カード」なのか「検索結果」なのか初見では分かりにくい。ギリシャ語入力ではInsightが出ないため、同じ「愛」でも日本語とギリシャ語で違う体験になることが説明されていない。

## GOOD

1. **Placeholder が3入力形式を提示**: 「愛・信じる・ἀγαπάω …」は日本語/エイリアス/ギリシャ語を一行で伝える
2. **テーマから探す（8件）**: 初心者の入口として機能。コンセプト探索の主要経路
3. **主要ラベル完全一致**: 愛・希望・救い・赦し・神・キリスト 全てのConceptが即発火
4. **エイリアス機能**: 「信じる」→信仰、「救われる」→救い、「悔い改める」→悔い改め が正常動作
5. **日本語サジェスト**: 「愛」入力でἀγαπάω候補が出る。日本語→ギリシャ語の橋渡し
6. **履歴の段階表示**: 初回は非表示、検索後に自動表示（空き状態を乱さない設計）
7. **文法パターン4件**: 高度ユーザー向けの入口として存在
8. **Empty State の情報階層**: 例→テーマ→文法パターン の順序が自然

## Recommended UX-7-2

### Phase UX-7-2-A: Zero-hit Recovery UI（最優先）

**対象**: `search-tool.html` の `zero-hits` 表示部分  
**変更内容**: 0件時に「テーマから探す」チップ（Top4件程度）と「ヒント」テキストを追加

```
∅
「神様」に該当する箇所がありませんでした

--- 提案 ---
テーマから探してみませんか:
[愛] [罪] [救い] [祈り]

ヒント: 「神様」→「神」、「お祈り」→「祈り」と入力してみてください
```

**実装規模**: 小（zero-hits innerHTML に静的または動的な recovery セクション追加）  
**変更禁止**: search-concepts.json / resolveConcept / 検索ロジック

---

### Phase UX-7-2-B: Strong番号検索 調査・修正

**手順**:
1. 本番ブラウザで G3056 を検索 → 結果件数を確認
2. 0件ならコード調査（searchStrong関数/インデックス構造）
3. 本番でも0件なら BLOCK として修正。動作するなら headless のタイミング問題として文書化

---

### Phase UX-7-2-C: 自然語表記 alias 追加（任意）

**対象**: `assets/data/search-concepts.json`  
**候補**:
- `god.aliases` に「神様」「主」を追加
- `prayer.aliases` に「お祈り」「お祈りする」を追加

**注意**: alias は「その語が当該 Concept 以外を指さない高信頼語のみ」（schema_note 準拠）。  
「主（しゅ）」は文脈によってキリストを指す場合もあるため慎重に判断する。
